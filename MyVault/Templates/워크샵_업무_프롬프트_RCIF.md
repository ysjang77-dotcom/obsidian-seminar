# 업무 프롬프트 워크샵 (RCIF)

> 목적: 본인의 실제 업무 2가지를 선택해, RCIF(역할/맥락/지시/형식)로 프롬프트를 완성하고 검증 계획까지 적는다.

## 업무 프롬프트 #1

### 작업 설명
(어떤 업무인지 1~2문장)

### 프롬프트

[R: 역할] R(Role): AI에게 역할/전문가 페르소나를 부여한다


[C: 맥락]  C(Context): 배경/목적/대상/제약/이전 작업 등 맥락을 제공한다


[I: 지시]  I(Instruction): 해야 할 일을 동사로 구체화하고, 수량/단계를 명시한다


[F: 형식] F(Format): 출력 형식(표/불릿/마크다운/JSON/템플릿)을 지정한다


### 검증 계획
1. 출처 요청: “근거/출처 링크(문서/논문/공식사이트)를 함께 줘”
2. 교차 검증: 2개 이상의 독립 출처로 확인
3. 전문가/원자료 검토: 중요한 내용은 최종 확인


### 3) 실전 기법 (필요할 때 추가)
- 예시 제공(Few-shot): 원하는 출력 예시 1~2개를 먼저 보여주기
- 단계별 요청: “먼저 분석 → 대안 3개 → 장단점 → 최종안”
- 반복 개선(Iterative): 1차 결과를 보고 구체 피드백으로 재요청
