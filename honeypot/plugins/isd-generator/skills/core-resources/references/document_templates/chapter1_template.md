# Chapter 1 Document Template

## 문서 구조 개요

제1장 개발 대상 및 필요성은 크게 3개의 절(Section)로 구성된다:
1. 개발 대상(Target) 정의
2. 기술분류 체계도(Technology tree) 및 개발 영역
3. 사업 필요성

각 섹션의 목적과 작성 가이드라인을 아래에 상세히 기술한다.

---

## 1. 개발 대상(Target) 정의

### 목적
연구개발의 핵심 대상과 목표를 명확히 정의하고, 기술적 가치와 차별성을 제시한다.

### 구성요소

#### 1-1. 개발 대상 (필수)
- 1-2문장으로 핵심 개발 목표 요약
- 세부기술 3개를 bullet point로 나열
- 사업개념도 이미지 위치 표시

```
### 개발 대상

[핵심 개발 목표를 1-2문장으로 요약. 동사형 어미 사용 (~개발, ~구축, ~확보)]

[세부기술 구분이 필요한 경우 아래와 같이 작성]

- (세부기술1) [세부기술명 및 간략 설명]
- (세부기술2) [세부기술명 및 간략 설명]
- (세부기술3) [세부기술명 및 간략 설명]

<사업개념도>

(기술이 주는 명백한 혜택) [핵심 가치 제안 - 구체적 수치 포함]

#### 1-2. 연구결과의 가치 및 성과지표 (필수)
- End Product와 경제/기술적 성과를 테이블로 정리
- F(First), B(Best), O(Only) 지표 활용

```

## 2. 기술분류 체계도(Technology tree) 및 개발 영역

### 목적
개발 대상 기술을 체계적으로 분류하고, 국내외 기술 수준 및 개발 역할을 정의한다.

### 작성 가이드

#### 2-1. 기술 영역별 설명 (선택)
- 각 기술 영역에 대한 설명 문단
- 세부기술 수에 따라 3-5개 기술 영역

```
### [기술 분류 제목]

#### [기술영역 1명]
- [기술 설명]
- [주요 개발 내용]

#### [기술영역 2명]
- [기술 설명]
- [주요 개발 내용]
```

#### 2-2. 기술분류 테이블 (필수)
- Depth 1/2/3 계층 구조
- 국내/국외 기술확보 수준 (O/△/x)
- 기술개발 영역 및 담당기관

```
### [기술 분류 제목]

| Depth 1 | Depth 2 | Depth 3 | 국내기술확보 | 국외기술확보 | 기술개발 영역(담당기관) |
|---------|---------|---------|-------------|-------------|----------------------|
| [대분류1] | [중분류1-1] | [소분류1-1-1] | O/△/x | O/△/x | [개발내용] (기관명) |
| | | [소분류1-1-2] | O/△/x | O/△/x | [개발내용] (기관명) |
| | [중분류1-2] | [소분류1-2-1] | O/△/x | O/△/x | [개발내용] (기관명) |
| [대분류2] | [중분류2-1] | [소분류2-1-1] | O/△/x | O/△/x | [개발내용] (기관명) |

<기술분류 체계도>
```

기술 수준 표시 기준:
- O: 기술 확보됨 (상용화 또는 실용화 단계)
- △: 일부 확보 또는 개발 중 (연구/시제품 단계)
- x: 미확보 (기초연구 또는 미착수)

## 작성 시 주의사항

1. 모든 주장에는 근거(데이터, 출처)를 제시
2. 이미지/도표는 `<이미지명, 출처: 기관명 (연도)>` 형식으로 위치 표시

4. VoC는 실제 기업명과 구체적 요구사항 포함
5. 기술 수준 비교 시 객관적 기준 제시 (O/△/x)
6. `**`볼드나 이모지 사용 금지
7. 헤더 계층 구조 준수 (#, ##, ###, ####)
