## 각 섹션별 작성 지침

### 1. 최종 연구 목표

#### 비전 선언문 작성법

형식: `[핵심 기술/방법론] 기반 [적용 대상]의 [목표 상태/기능] 개발`

예시:
- 필드환경 특화 피지컬 AI 기반 이종 산업기계의 범용 자율화 플랫폼 개발
- 생성형 AI 기반의 설계 특화 AI Agent 및 협업 플랫폼 기술 개발을 통한 설계 엔지니어링 자율화 기술 개발
- 다수/다종의 건설기계가 사람의 개입 없이 동적 환경에 적응하며 자율적으로 협력 작업을 수행하는 AI 건설기계 다개체 협업플랫폼 개발

#### 단계별 목표 작성법

1단계 (1-3차년도): 기반기술 확보, 요소기술 개발, 시제품 개발
- 예: 필드환경 자율주행/작업 AI 모델 개발 및 레트로핏 자율화 모듈 시제품 개발

2단계 (4-5차년도): 고도화, 통합, 실증, 상용화 검증
- 예: AI 고도화 및 다기종 범용 레트로핏 키트 개발, 3대 분야 현장 실증

### 2. 세부기술 개요

#### 세부기술 분류 패턴

패턴 A (기능별 분류):
- 세부기술1: 인식/센싱 기술
- 세부기술2: 계획/의사결정 기술
- 세부기술3: 제어/실행 기술

패턴 B (계층별 분류):
- 세부기술1: 핵심 AI/알고리즘
- 세부기술2: 하드웨어/모듈
- 세부기술3: 플랫폼/시스템

패턴 C (대상별 분류):
- 세부기술1: 대상 A 특화 기술
- 세부기술2: 대상 B 특화 기술
- 세부기술3: 통합/공통 기술

#### 세부기술 항목 작성법

각 세부기술당 4-5개 핵심 기술 항목 나열:

```
#### (세부기술1) 필드 특화 자율주행/자율작업 AI 개발
> 대상작업: 토공작업(굴착/적재), 밭경작(정지/파종), 재난수색(이동/탐지)

- 필드환경 특화 자율주행/자율작업 AI 모델 개발 (오픈소스 기반 자체 개발)
- AI 출력-장비 제어 연계 브릿지 제어 시스템 개발
- LLM 기반 자연어 작업 지시 시스템 개발
- OOD 탐지 및 Fail-Safe 안전 시스템 개발
```

#### 성과지표 작성법

```
| 기술개발 영역 | 연도 | 성과지표 |
|:---|:---:|:---|
| (1) [영역명] | 2027 | - 성과물1<br>- 성과물2 |
| | 2028 | - 성과물1<br>- 성과물2 |
| | 2029 | - 성과물1<br>- 성과물2 |
| | 2030 | - 성과물1<br>- 성과물2 |
| | 2031 | - 성과물1<br>- 성과물2 |
```

성과물 명명 규칙:
- SW: ~알고리즘 SW, ~시스템 SW, ~플랫폼
- HW: ~시제품, ~모듈, ~장비
- 문서: ~설계서, ~보고서, ~매뉴얼
- 데이터: ~데이터셋, ~DB, ~모델

버전 표기 규칙 (v1.0, v2.0 등):
- **버전 표기는 동일 산출물이 연차별로 발전할 때만 사용**
- 단일 버전만 존재하는 산출물에는 버전 번호를 붙이지 않음
- 올바른 예: "범용 모델" (단일) → 버전 없이 명명
- 올바른 예: "범용 모델 v1.0" (2030년) → "범용 모델 v2.0" (2031년) - 연차별 발전 시 버전 표기
- 잘못된 예: "통합 AI v2.0" (v1.0 없이 갑자기 v2.0 사용 금지)
- 잘못된 예: "이기종 범용 모델 v1.0" (다음 연차에 v2.0이 없으면 v1.0도 사용 금지)
