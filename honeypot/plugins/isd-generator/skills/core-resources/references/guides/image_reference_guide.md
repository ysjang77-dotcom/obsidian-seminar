# Image Reference Guide

## 개요

Chapter 1 문서에서 사용되는 이미지/도표의 수집, 출처 표기, 활용 방법을 정의한다. 가능한 한 실제 데이터와 검증된 출처를 사용하며, 모든 이미지에 출처를 명시한다.

---

## 이미지/도표 유형

### 1. 사업 개념도
- 위치: 1. 개발 대상 - 개발 대상 섹션
- 목적: 전체 사업의 구조와 범위를 시각화
- 구성요소:
  - 최종 목표 (상단)
  - 3개 세부기술 (중앙)
  - 기대효과/적용분야 (하단)
- 권장 형식: 계층 구조도, 흐름도

### 2. 개발 개념도
- 위치: 1. 개발 대상 - 개발 개념 섹션
- 목적: 기술 개발 접근 방식과 시스템 아키텍처 시각화
- 구성요소:
  - 입력/데이터 (좌측)
  - 핵심 기술 요소 (중앙)
  - 출력/결과물 (우측)
  - 기존 인프라 연계 (하단)
- 권장 형식: 블록 다이어그램, 시스템 아키텍처

### 3. 기술분류 체계도
- 위치: 2. 기술분류 체계도 섹션
- 목적: 개발 기술의 계층적 분류 시각화
- 구성요소:
  - Depth 1 (대분류)
  - Depth 2 (중분류)
  - Depth 3 (소분류)
  - 기술 수준 표시
- 권장 형식: Technology Tree, 마인드맵

### 4. 시장 규모/전망 도표
- 위치: 3. 사업 필요성 - 추진배경 및 필요성 섹션
- 목적: 관련 시장의 규모와 성장 전망 제시
- 출처 요구: 반드시 실제 시장조사 보고서 인용
- 권장 출처:
  - MarketsandMarkets
  - Frost & Sullivan
  - Statista
  - 산업연구원 (KIET)
  - 한국과학기술기획평가원 (KISTEP)

### 5. 정책 로드맵/연계도
- 위치: 3. 사업 필요성 - 정부정책과 부합성 섹션
- 목적: 정부 정책과 과제의 연계성 시각화
- 출처 요구: 정부 발표 자료 인용
- 권장 출처:
  - 과학기술정보통신부
  - 산업통상자원부
  - 국가과학기술자문회의
  - NTIS (국가과학기술지식정보서비스)

### 6. 기술 수준 비교 도표
- 위치: 3. 사업 필요성 - 추진배경 및 필요성 섹션
- 목적: 국내외 기술 수준 격차 시각화
- 출처 요구: 기술수준평가 보고서 인용
- 권장 출처:
  - KISTEP 기술수준평가
  - 산업기술평가관리원 (KEIT)
  - 국가과학기술자문회의 보고서

## 출처 표기 형식

### 기본 형식
```
[이미지/도표명, 출처: 기관명 (연도)]
```

### 상세 표기 (권장)
```
[이미지/도표명]
출처: [보고서명/문서명], [발행기관], [연도]
URL: [접근 가능한 URL]
```

### 예시

#### 정부 정책 자료
```
[12대 국가전략기술 로드맵]
출처: 제1차 국가전략기술 육성 기본계획, 과학기술정보통신부, 2024
URL: https://www.msit.go.kr/...
```

#### 시장 조사 보고서
```
[글로벌 AI 시장 규모 전망]
출처: AI Market Report 2024, MarketsandMarkets, 2024
URL: https://www.marketsandmarkets.com/...
```

#### 기관 발표 자료
```
[KIMM 중점육성 8대 분야]
출처: 한국기계연구원 중장기발전계획(KIMM2030), 한국기계연구원, 2024
```

## 이미지 수집 프로세스

### Step 1: 웹 검색으로 실제 도표 탐색

검색어 예시:
```
"[분야] 시장 규모 그래프 site:kistep.re.kr"
"[기술] 기술수준평가 site:keit.re.kr"
"[분야] 정책 로드맵 site:msit.go.kr"
"[분야] market size chart filetype:pdf"
```

### Step 2: 출처 검증

필수 확인 사항:
- 발행 기관의 신뢰성 (정부기관, 공인 연구기관, 글로벌 조사기관)
- 발행 연도 (최근 2-3년 이내 권장)
- 데이터의 구체성 (수치, 연도, 단위 포함 여부)

### Step 3: 출처 기록

수집된 모든 이미지/도표에 대해 다음 정보 기록:
- 도표 제목
- 발행 기관
- 문서/보고서명
- 발행 연도
- URL (가능한 경우)
- 핵심 데이터 요약

## 이미지 생성 옵션 (실제 도표 미확보 시)

### 옵션 A: 데이터 기반 이미지 프롬프트

검증된 데이터를 기반으로 이미지 생성 프롬프트 제공:

```
[이미지 생성 프롬프트]
제목: [도표 제목]
유형: [막대그래프/원형차트/흐름도 등]
데이터:
- 항목1: [수치] (출처: [기관명], [연도])
- 항목2: [수치] (출처: [기관명], [연도])
스타일: Clean corporate style, monochromatic blue (#1a365d, #2c5282)
```

### 옵션 B: 구조적 다이어그램 프롬프트

개념/구조 시각화를 위한 프롬프트:

```
[이미지 생성 프롬프트]
제목: [다이어그램 제목]
유형: [계층구조도/네트워크/블록다이어그램]
구성요소:
- 레벨1: [요소들]
- 레벨2: [요소들]
- 연결관계: [설명]
스타일: Minimal, white background, blue accent (#2c5282)
```

```markdown

### 1. [도표명]
- 유형: [막대그래프/원형차트/흐름도]
- 출처: [보고서명], [기관명], [연도]
- URL: [URL]
- 핵심 데이터:
  - [데이터1]
  - [데이터2]
- 문서 내 활용 위치: [섹션명]

### 2. [도표명]
...
```

## 주의사항

1. 출처 미상의 이미지는 사용하지 않음
2. 오래된 데이터(5년 이상)는 최신 데이터로 업데이트 권장
3. 저작권 문제가 있을 수 있는 상업 보고서는 인용 형식으로 처리
4. 자체 제작 이미지도 "자체 제작" 또는 "[기관명] 자체 작성"으로 표기
5. 정부 발표 자료는 공개 자료만 활용 (비공개/내부 자료 제외)
