# 이미지 프롬프트 작성 가이드

본 가이드는 국책과제 연구계획서의 이미지 생성을 위한 프롬프트 작성 규칙을 정의한다.

---

## 1. 기본 요구사항

### 이미지 유형
- PPT 슬라이드 스타일의 인포그래픽
- 정부 국가연구개발 과제 발표자료 수준의 전문적 도식
- 가로형(16:9 비율), 고해상도, 벡터 그래픽 스타일
- 한글로 작성됨

### 디자인 원칙
- 과도한 색상 사용 금지 (4색 이하로 제한)
- 지나치게 화려하지 않으나 보기 편한 디자인
- 시인성과 구성력, 몰입도를 갖춘 정부 공식 발표자료 수준
- 정보의 구조화로 이해하기 편한 구성

---

## 2. 색상 팔레트 (4색 제한)

### 기본 파란색 계열 팔레트

| 역할 | 색상명 | HEX 코드 | RGB | 사용처 |
|------|--------|----------|-----|--------|
| 주조색 | 진한 네이비 | #1E3A5F | 30, 58, 95 | 제목, 핵심 박스, 후반기 요소 |
| 보조색 | 청록색 | #4A90A4 | 74, 144, 164 | 화살표, 프로세스, 전반기 요소 |
| 강조색 | 진한 녹색 | #2E7D5A | 46, 125, 90 | 성과/목표 달성, 완료 표시 |
| 배경색 | 밝은 회색 | #F5F7FA | 245, 247, 250 | 배경, 보조 박스 |

### 색상 사용 규칙

- 주조색: 대제목, 핵심 박스 배경, 중요 정보
- 보조색: 연결선, 화살표, 프로세스 표시, 초기 단계
- 강조색: 성과, 달성 목표, 완료 항목, 체크마크
- 배경색: 전체 배경, 보조 정보 박스

## 3. 텍스트 규격

### 폰트 크기 및 굵기

| 요소 | 크기 | 굵기 | 색상 | 용도 |
|------|------|------|------|------|
| 대제목 | 24pt | Bold | 네이비(#1E3A5F) | 이미지 상단 타이틀 |
| 섹션 제목 | 18pt | Bold | 네이비(#1E3A5F) | 영역 구분 제목 |
| 소제목 | 14pt | SemiBold | 네이비(#1E3A5F) | 박스 내 제목 |
| 본문 | 11-12pt | Regular | 진회색(#333333) | 일반 텍스트 |
| 강조텍스트 | 12pt | Bold | 녹색(#2E7D5A) | 핵심 키워드 |
| 라벨 | 10pt | Regular | 진회색(#666666) | 보조 설명 |

### 텍스트 배치 원칙

- 가독성 우선: 충분한 여백과 줄간격 확보
- 정보 계층화: 중요도에 따른 크기/굵기 차등
- 좌측 정렬 기본: 본문 텍스트는 좌측 정렬
- 중앙 정렬: 제목, 박스 내 핵심 텍스트

## 4. 그래픽 요소 스타일

### 박스 스타일

```
- 모든 박스: 둥근 모서리 (border-radius: 8px)
- 그림자: 연한 드롭섀도우 (2px blur, 10% opacity)
- 테두리: 1-2px, 해당 색상
- 배경: 흰색(#FFFFFF) 또는 연한 색상
```

### 화살표 및 연결선

```
- 화살표 두께: 3px
- 화살표 색상: 보조색(청록색) 기본
- 방향 화살표: 삼각형 끝
- 점선: 관계 표시용 (2px dash)
```

### 구분선

```
- 두께: 1px
- 색상: 연회색(#DDDDDD)
- 섹션 간 구분용
```

### 아이콘/심볼

```
- 체크마크(✓): 녹색(#2E7D5A)
- 불릿 포인트: 원형(●) 또는 다이아몬드(◆)
- 숫자 배지: 원형 배경 + 흰색 숫자
```

### 16:9 비율 권장 배치

- 상단 타이틀: 전체 높이의 8-10%
- 메인 컨텐츠: 전체 높이의 80-85%
- 하단 영역: 전체 높이의 5-10% (선택)
- 좌우 여백: 전체 너비의 3-5%

## 7. 프롬프트 작성 순서

### Step 1: 캡션 분석
- 캡션명에서 이미지 유형 파악
- 해당 문서 섹션에서 포함할 내용 추출

### Step 2: 기본 구조 작성
```markdown
[이미지 생성 프롬프트]

## 이미지 유형
[기본 요구사항 명시]

## 색상 팔레트 (4색 제한)
[4색 팔레트 명시]
```

### Step 3: 레이아웃 상세화
```markdown
## 전체 레이아웃 구성
[ASCII 다이어그램으로 레이아웃 표현]

## 영역별 상세 구성
[각 영역의 내용과 스타일 명시]
```

### Step 4: 텍스트 명시
```markdown
## 반드시 포함할 텍스트 (한글)
- "텍스트1"
- "텍스트2"
- ...
```

### Step 5: 그래픽 요소 정의
```markdown
## 그래픽 요소
[박스, 화살표, 아이콘 스타일 명시]

## 텍스트 규격
[폰트 크기, 굵기, 색상 명시]
```

## 9. 참고 데이터 출처 작성 가이드

프롬프트의 모든 정보는 출처를 명시하여 검증성과 수정 용이성을 확보한다.

### 9.1 테이블 구조 정의

각 카테고리별로 별도 테이블을 작성한다:

```markdown
### [카테고리명] 데이터

| 항목 | 내용 | 출처 |
|------|------|------|
| [항목명] | [구체적 내용] | [문서명]:라인[N] |
```

### 9.2 출처 표기법

```
형식: [문서명]:라인[N] 또는 [문서명]:섹션[섹션명]

예시:
- 연구목표및내용.md:라인45
- 추진전략.md:섹션3-2
- chapter_3/output.md:라인120-135
```

### 9.3 필수 카테고리 (최소 3개)

| 카테고리 | 포함 내용 | 필수 열 |
|----------|----------|---------|
| 연차별 계획 | 연차, 주요 내용, 목표 | 연차, 내용, 출처 |
| 기술 지표 | 목표 수치, 단위, 달성 시점 | 지표, 수치, 단위, 출처 |
| 기관/협력 | 기관명, 역할, 담당 내용 | 기관, 역할, 출처 |
| 바이오마커/기술 | 명칭, 유형, 비고 | 항목, 내용, 출처 |
| 성과 목표 | 성과물, 수량, 달성 연차 | 성과, 목표, 출처 |

### 9.4 데이터 추출 규칙

1. **수치 데이터**: 정확한 숫자와 단위 기록 (예: "90% 이상", "fM 수준")
2. **기술 용어**: 원문 그대로 인용 (예: "CRISPR-Cas13a")
3. **기관명**: 공식 명칭 사용 (예: "[주관기관]", "[협력기관1]")
4. **연차 표기**: 일관된 형식 (예: "1년차", "전반기(1-3년차)")

## 10. 그림 구성 핵심 포인트 작성법

이미지의 핵심 메시지와 시각적 흐름을 명확히 정의한다.

### 10.1 필수 항목 (5개)

```markdown
## 그림 구성 핵심 포인트

1. **[영역 1명]**: [해당 영역이 표현해야 할 핵심 내용과 방식]
2. **[영역 2명]**: [해당 영역이 표현해야 할 핵심 내용과 방식]
3. **[영역 3명]**: [해당 영역이 표현해야 할 핵심 내용과 방식]
4. **시각적 흐름**: [독자가 정보를 읽는 순서]
5. **핵심 메시지**: [이 그림이 전달해야 할 핵심 메시지]
```

### 10.2 영역별 설명 방식

| 영역 유형 | 설명 포함 내용 | 예시 |
|----------|---------------|------|
| 좌측 영역 | 무엇을 보여주는지 + 표현 방식 | "5단계 프로세스를 세로 플로우차트로 직관적 표현" |
| 우측 영역 | 무엇을 보여주는지 + 강조점 | "5년 타임라인을 단계별로 구분하여 명확히 제시" |
| 하단 영역 | 무엇을 보여주는지 + 구조 | "기관 협력 구조를 중심-주변 배치로 표현" |

### 10.3 시각적 흐름 정의

```markdown
시각적 흐름 예시:
- 상단(제목) → 좌측(프로세스) → 우측(일정) → 하단(협력) 순서
- 중심(핵심 목표) → 주변(세부 기술) → 하단(기대효과) 순서
- 좌에서 우(시간 흐름) + 상에서 하(상세화) 동시 진행
```

### 10.4 핵심 메시지 작성법

1-2문장으로 이 그림을 본 독자가 얻어야 할 핵심 인사이트 기술:

```markdown
예시:
- "5년간 CRISPR 기술 개발부터 현장진단 플랫폼까지 단계적 완성"
- "[주관기관] 중심의 산학연 협력으로 전주기 진단 시스템 구축"
- "전반기 기술개발, 후반기 임상적용으로 체계적 추진"
```

## 11. 정보 계층 구조 정의

이미지 내 정보의 중요도를 3단계로 구분하여 시각적 강조 수준을 결정한다.

### 11.1 계층 분류 기준

| 계층 | 중요도 | 시각적 표현 | 포함 내용 |
|------|--------|------------|----------|
| Primary (1차) | 최고 | 큰 크기, 진한 색상, 굵은 텍스트 | 제목, 최종 목표, 핵심 성과 |
| Secondary (2차) | 중간 | 중간 크기, 보조 색상, 일반 텍스트 | 세부 단계, 연차별 내용, 기관명 |
| Tertiary (3차) | 보조 | 작은 크기, 연한 색상, 가는 텍스트 | 세부 설명, 비고, 출처 |

### 11.2 계층별 색상 매핑

```markdown
Primary   → 네이비(#1E3A5F) 배경 + 흰색 텍스트
Secondary → 청록(#4A90A4) 테두리 + 네이비 텍스트
Tertiary  → 연회색(#F5F7FA) 배경 + 진회색 텍스트
성과/강조  → 녹색(#2E7D5A) 테두리/배경
```

### 11.3 계층별 텍스트 크기

```markdown
Primary   → 18-24pt, Bold
Secondary → 12-16pt, SemiBold
Tertiary  → 10-11pt, Regular
```

### 11.4 정보 계층 작성 예시

```markdown
## 정보 계층 구조

- **Primary (1차)**:
  - "전주기 방광암 진단 플랫폼 구축" (최종 목표)
  - "5년차 통합 플랫폼 완성" (최종 성과)

- **Secondary (2차)**:
  - STEP 1~5 (5단계 프로세스)
  - 1-5년차 연차별 내용
  - [주관기관], [협력기관1], [협력기관2] (기관명)

- **Tertiary (3차)**:
  - 세부 기술 설명 (CRISPR-Cas13a 작동 원리)
  - 수치 데이터 (민감도: fM 수준)
  - 출처 정보
```

## 12. 렌더링 금지 패턴 (중요)

다음 패턴은 AI가 이미지에 직접 렌더링하므로, 프롬프트에서 **표시할 텍스트**와 **스타일 지시**를 반드시 분리하여 작성해야 한다.

### 12.1 금지 패턴 목록

| 금지 패턴 | 문제점 | 올바른 방식 |
|----------|--------|------------|
| `"제목" (24pt)` | "24pt"가 이미지에 렌더링됨 | 텍스트 규격 섹션에 별도 명시 |
| `"텍스트" ← 네이비 배경` | "← 네이비 배경"이 렌더링됨 | 색상 팔레트/그래픽 요소 섹션에 별도 명시 |
| `[설명 텍스트]` | 대괄호 내용이 그대로 렌더링됨 | 작성 지침 섹션으로 분리 |
| `(강조)`, `(크게)` | 괄호 내용이 렌더링됨 | 그래픽 요소 섹션에 별도 명시 |
| `48pt Bold 주황색` | 폰트 사양이 텍스트로 렌더링됨 | 텍스트 규격 테이블에 별도 정의 |

### 12.2 잘못된 작성 예시

```
❌ 잘못된 방식 (이 텍스트들이 이미지에 그대로 렌더링됨):

┌───────────────┐
│   평균 66세   │ ← 주황색, 48pt Bold
│              │   대형 숫자
└───────────────┘

[반 이상이 70세 이상임을 시각적으로 강조]

"농업현장 인력 위기" (24pt, Bold, 중앙정렬)
```

### 12.3 올바른 작성 예시

```
✅ 올바른 방식 (표시 텍스트와 스타일 지시 분리):

## 반드시 포함할 텍스트 (이미지에 표시됨)
- "평균 66세"
- "50.8%"
- "농업현장 인력 위기"

## 텍스트 규격 (스타일 정의용 - 렌더링 안됨)
| 요소 | 크기 | 굵기 | 색상 |
|------|------|------|------|
| 대제목 | 24pt | Bold | 흰색 |
| 통계 숫자 | 48pt | Bold | 주황색(#E67E22) |

## 작성 지침 (렌더링 안됨 - 의도 전달용)
- 통계 숫자를 크게 강조하여 시각적 임팩트 부여
- 반 이상이 70세 이상임을 진행률 바로 시각적 표현
- 대제목은 중앙 정렬로 배치

## ASCII 레이아웃 (구조만 표현 - 메타 정보 제외)
┌───────────────┐
│   평균 66세   │
│              │
└───────────────┘
```

### 12.4 ASCII 레이아웃 작성 규칙

1. **박스 내부**: 실제 표시될 텍스트만 기재 (스타일 정보 제외)
2. **화살표 주석 금지**: `← 설명` 형식 사용하지 않음
3. **색상/크기 주석 금지**: `(네이비)`, `(크게)` 등 제외
4. **스타일 정보는 별도 섹션에서 정의**: "텍스트 규격", "그래픽 요소" 섹션 활용

### 12.5 프롬프트 섹션별 역할 구분

| 섹션 | 내용 | 이미지 렌더링 |
|------|------|--------------|
| 반드시 포함할 텍스트 | 실제 표시될 텍스트 목록 | ✅ 렌더링됨 |
| ASCII 레이아웃 | 구조와 위치 (텍스트만) | ✅ 렌더링됨 |
| 텍스트 규격 | 폰트 크기/굵기/색상 정의 | ❌ 렌더링 안됨 |
| 그래픽 요소 | 박스/화살표 스타일 정의 | ❌ 렌더링 안됨 |
| 작성 지침 | 의도/강조점 설명 | ❌ 렌더링 안됨 |
| 색상 팔레트 | HEX 코드 및 용도 정의 | ❌ 렌더링 안됨 |
