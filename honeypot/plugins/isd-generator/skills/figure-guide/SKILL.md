# 이미지 생성 프롬프트 작성 가이드

국책과제 연구계획서의 이미지 생성을 위한 프롬프트 작성 규칙을 정의한다.

---

## 1. 기본 요구사항

### 이미지 유형
- PPT 슬라이드 스타일의 인포그래픽
- 정부 국가연구개발 과제 발표자료 수준의 전문적 도식
- 가로형(16:9 비율), 고해상도, 벡터 그래픽 스타일
- 한글로 작성됨

### 디자인 원칙
- 과도한 색상 사용 금지 (4색 이하로 제한)
- 지나치게 화려하지 않으나 보기 편한 디자인
- 시인성과 구성력, 몰입도를 갖춘 정부 공식 발표자료 수준
- 정보의 구조화로 이해하기 편한 구성

## 2. 색상 팔레트 (4색 제한)

### 기본 파란색 계열 팔레트

| 역할 | 색상명 | HEX 코드 | RGB | 사용처 |
|------|--------|----------|-----|--------|
| 주조색 | 진한 네이비 | #1E3A5F | 30, 58, 95 | 제목, 핵심 박스, 후반기 요소 |
| 보조색 | 청록색 | #4A90A4 | 74, 144, 164 | 화살표, 프로세스, 전반기 요소 |
| 강조색 | 진한 녹색 | #2E7D5A | 46, 125, 90 | 성과/목표 달성, 완료 표시 |
| 배경색 | 밝은 회색 | #F5F7FA | 245, 247, 250 | 배경, 보조 박스 |

### 색상 사용 규칙

- 주조색: 대제목, 핵심 박스 배경, 중요 정보
- 보조색: 연결선, 화살표, 프로세스 표시, 초기 단계
- 강조색: 성과, 달성 목표, 완료 항목, 체크마크
- 배경색: 전체 배경, 보조 정보 박스

## 3. 텍스트 규격

### 폰트 크기 및 굵기

| 요소 | 크기 | 굵기 | 색상 | 용도 |
|------|------|------|------|------|
| 대제목 | 24pt | Bold | 네이비(#1E3A5F) | 이미지 상단 타이틀 |
| 섹션 제목 | 18pt | Bold | 네이비(#1E3A5F) | 영역 구분 제목 |
| 소제목 | 14pt | SemiBold | 네이비(#1E3A5F) | 박스 내 제목 |
| 본문 | 11-12pt | Regular | 진회색(#333333) | 일반 텍스트 |
| 강조텍스트 | 12pt | Bold | 녹색(#2E7D5A) | 핵심 키워드 |
| 라벨 | 10pt | Regular | 진회색(#666666) | 보조 설명 |

### 텍스트 배치 원칙

- 가독성 우선: 충분한 여백과 줄간격 확보
- 정보 계층화: 중요도에 따른 크기/굵기 차등
- 좌측 정렬 기본: 본문 텍스트는 좌측 정렬
- 중앙 정렬: 제목, 박스 내 핵심 텍스트

## 4. 그래픽 요소 스타일

### 박스 스타일

```
- 모든 박스: 둥근 모서리 (border-radius: 8px)
- 그림자: 연한 드롭섀도우 (2px blur, 10% opacity)
- 테두리: 1-2px, 해당 색상
- 배경: 흰색(#FFFFFF) 또는 연한 색상
```

### 화살표 및 연결선

```
- 화살표 두께: 3px
- 화살표 색상: 보조색(청록색) 기본
- 방향 화살표: 삼각형 끝
- 점선: 관계 표시용 (2px dash)
```

### 구분선

```
- 두께: 1px
- 색상: 연회색(#DDDDDD)
- 섹션 간 구분용
```

### 아이콘/심볼

```
- 체크마크(✓): 녹색(#2E7D5A)
- 불릿 포인트: 원형(●) 또는 다이아몬드(◆)
- 숫자 배지: 원형 배경 + 흰색 숫자
```

### 16:9 비율 권장 배치

- 상단 타이틀: 전체 높이의 8-10%
- 메인 컨텐츠: 전체 높이의 80-85%
- 하단 영역: 전체 높이의 5-10% (선택)
- 좌우 여백: 전체 너비의 3-5%

## 6. 캡션 유형별 분류

### 분류 기준

캡션명에 포함된 키워드를 기반으로 9가지 유형으로 분류한다.

### 유형별 키워드 및 이미지 스타일

| 유형 | 우선순위 | 키워드 | 이미지 스타일 |
|------|----------|--------|--------------|
| 비전/목표 | 1 | 목표, 비전, 개념, 최종목표, 연구목표 | 계층 구조도, 피라미드 |
| 로드맵 | 2 | 로드맵, 일정, 타임라인, 마일스톤 | 타임라인, 간트 차트 |
| 연차별 | 3 | 연차별, 년차별, 단계별, 연도별, 1단계, 2단계 | 테이블, 타임라인 |
| 추진체계 | 4 | 추진체계, 조직, 역할, R&R, 거버넌스 | 조직도 |
| 투자계획 | 5 | 투자, 예산, 재원, 비용, 자금 | 막대 그래프, 파이 차트 |
| 협력체계 | 6 | 협력, 네트워크, 파트너, 컨소시엄 | 네트워크 다이어그램 |
| 사업화 | 7 | 사업화, 상용화, 활용, 적용, 시장진출 | 흐름도, 로드맵 |
| 개념도 | 8 | 개념, 구조, 아키텍처, 시스템, 프레임워크 | 블록 다이어그램 |
| 기술분류 | 9 | 기술분류, 체계, 분류, 트리, 계통 | Technology Tree |

### 분류 알고리즘

```
1. 캡션명을 소문자로 변환
2. 각 유형의 키워드 목록과 매칭
3. 먼저 매칭되는 유형으로 분류
4. 매칭 안 되면 "개념도"(기본값)로 분류
```

## 8. 프롬프트 작성 순서

### Step 1: 캡션 분석
- 캡션명에서 이미지 유형 파악
- 해당 문서 섹션에서 포함할 내용 추출

### Step 2: 기본 구조 작성
```markdown
[이미지 생성 프롬프트]

## 이미지 유형
[기본 요구사항 명시]

## 색상 팔레트 (4색 제한)
[4색 팔레트 명시]
```

### Step 3: 레이아웃 상세화
```markdown
## 전체 레이아웃 구성
[ASCII 다이어그램으로 레이아웃 표현]

## 영역별 상세 구성
[각 영역의 내용과 스타일 명시]
```

### Step 4: 텍스트 명시
```markdown
## 반드시 포함할 텍스트 (한글)
- "텍스트1"
- "텍스트2"
- ...
```

### Step 5: 그래픽 요소 정의
```markdown
## 그래픽 요소
[박스, 화살표, 아이콘 스타일 명시]

## 텍스트 규격
[폰트 크기, 굵기, 색상 명시]
```

## 9. 렌더링 금지 패턴 (중요)

다음 패턴은 AI가 이미지에 직접 렌더링하므로, 프롬프트에서 **표시할 텍스트**와 **스타일 지시**를 반드시 분리하여 작성해야 한다.

### 금지 패턴 목록

| 금지 패턴 | 문제점 | 올바른 방식 |
|----------|--------|------------|
| `"제목" (24pt)` | "24pt"가 이미지에 렌더링됨 | 텍스트 규격 섹션에 별도 명시 |
| `"텍스트" ← 네이비 배경` | "← 네이비 배경"이 렌더링됨 | 색상 팔레트/그래픽 요소 섹션에 별도 명시 |
| `[설명 텍스트]` | 대괄호 내용이 그대로 렌더링됨 | 작성 지침 섹션으로 분리 |
| `(강조)`, `(크게)` | 괄호 내용이 렌더링됨 | 그래픽 요소 섹션에 별도 명시 |
| `48pt Bold 주황색` | 폰트 사양이 텍스트로 렌더링됨 | 텍스트 규격 테이블에 별도 정의 |

### 올바른 작성 예시

```
✅ 올바른 방식 (표시 텍스트와 스타일 지시 분리):

## 반드시 포함할 텍스트 (이미지에 표시됨)
- "평균 66세"
- "50.8%"
- "농업현장 인력 위기"

## 텍스트 규격 (스타일 정의용 - 렌더링 안됨)
| 요소 | 크기 | 굵기 | 색상 |
|------|------|------|------|
| 대제목 | 24pt | Bold | 흰색 |
| 통계 숫자 | 48pt | Bold | 주황색(#E67E22) |

## 작성 지침 (렌더링 안됨 - 의도 전달용)
- 통계 숫자를 크게 강조하여 시각적 임팩트 부여
- 반 이상이 70세 이상임을 진행률 바로 시각적 표현
- 대제목은 중앙 정렬로 배치
```
