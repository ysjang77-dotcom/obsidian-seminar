# Gov 스타일 테마 팔레트

정부/공공기관 PPT 슬라이드를 위한 9가지 무드 테마. 각 테마는 공식 문서와 발표 자료에 적합한 4색 팔레트를 제공합니다.

**시각 정체성**: "정부 백서처럼 — 굵은 테두리 박스에 번호가 매겨진 체계적 격자"

## 렌더링 스타일 (Rendering Style)

이 스타일의 시각적 DNA를 정의합니다. prompt-designer가 INSTRUCTION/CONFIGURATION 블록 생성 시 반드시 참조해야 합니다.

| 차원 | 지시 사항 |
|------|-----------|
| **서피스 (박스/패널)** | 2px 실선 테두리 + 내부 흰색 채움. 모든 정보 단위가 직사각형 박스 안에 담긴다. 박스 헤더 영역은 주조색 배경 + 흰색 글씨. 본문 영역은 흰색 배경 + 검정 글씨. |
| **배경** | 연한 그레이(#F5F7FA) 배경 + 상단에 주조색 가로 띠(배너). 배너 안에 문서 제목을 배치한다. 하단에 얇은 주조색 구분선. 공문서 헤더/푸터 구조. |
| **코너/엣지** | 완전 직각(0px radius). 모든 박스, 표, 구분선이 90도 직각. 라운딩 금지. 정부 문서의 엄격한 격식을 표현한다. |
| **연결선** | 2px 실선 + 채운 삼각형 화살표(▶). 굵고 명확하게 표현. 연결선 위에 라벨 텍스트 허용. 점선은 "계획/예정" 의미로만 사용한다. |
| **시각 장식** | 단색 플랫 아이콘(채워진 원 안에 흰색 심볼). 표(table)를 적극 활용한다. 항목 앞에 ● 또는 ■ 불릿. 번호 뱃지(①②③) 사용. |
| **타이포그래피** | 3단 위계(대/중/소) 균등 분포. 제목은 볼드, 박스 헤더는 세미볼드, 본문은 레귤러. 모든 텍스트 좌측 정렬. 숫자는 본문과 동일 크기. 일괄 정돈된 느낌. |
| **공간 구성** | 엣지-투-엣지 격자 + 20% 여백. 화면을 최대한 활용. 상단 배너 + 본문 그리드 + 하단 주석의 3단 구조. 2×2, 2×3, 3×3 격자 배치를 빈번하게 사용. |
| **시각 메타포** | 플랫 인포그래픽(Flat Infographic). 채워진 아이콘, 단색 차트 바, 표, 조직도. 2D 평면적 표현. 입체감이나 그라데이션을 최소화한다. 공공기관 보고서에서 볼 법한 정형화된 시각 요소. |

### 기관 식별 요소 금지 (CRITICAL)

이 테마는 **정부 발표용 스타일**이지, 정부에서 제작한 자료가 아닙니다. 따라서:

- 기관 로고, 기관 마크, 부처 심볼을 포함하지 않는다
- 특정 기관명, 부처명을 제목이나 배너에 고정 배치하지 않는다
- "OO부", "OO청", "OO위원회" 등 기관 식별 요소를 시각 장식으로 사용하지 않는다
- 태극기, 정부 문양 등 국가 상징을 포함하지 않는다

## Contrast 레이아웃 하단 영역 규칙

Contrast(대비) 레이아웃 사용 시, 화면은 3개 영역으로 나뉩니다:

| 영역 | 배경 처리 | 설명 |
|------|-----------|------|
| 좌측 As-Is (현황) | 어두운 톤 배경 (보조색 10% opacity 또는 #E8EAED) | 현재 상태/문제 영역 |
| 우측 To-Be (미래상) | 밝은 톤 배경 (#F5F7FA 또는 강조색 5% opacity) | 목표/솔루션 영역 |
| **하단 (기대효과/결론)** | **주조색 5-8% opacity 단일 배경** | 좌우 대비와 독립된 통합 영역 |

**핵심 규칙**:
- 하단 영역(기대효과, 결론, 종합 성과 등)은 좌우 대비 배경을 **상속하지 않는다**
- 하단은 **단일 통합 배경색**으로 처리하여 시각적 완결성을 확보한다
- 권장 하단 배경: 주조색을 5-8% opacity로 적용한 연한 톤 (예: 주조색이 #2C6AA0이면 → #EDF2F7 정도)
- 하단 영역 내 박스/항목은 좌우 균등 배치하여 정돈된 느낌을 유지한다

| 역할 라벨 | `Main Title`, `Sub-header` | INSTRUCTION에서만 사용 |
| 기관 로고/마크 | `[기관 로고]`, `정부 마크`, `부처 심볼` | 특정 기관 식별 요소 |

| 규칙 | 설명 |
|------|------|
| **플레이스홀더 금지** | `[Image 1]`, `[Image 2]`, `[사진]`, `[아이콘]` 등 대괄호 플레이스홀더가 이미지에 렌더링되면 안 된다. 모든 시각 요소는 구체적으로 기술하여 Gemini가 실제로 렌더링하도록 한다 |
| **깨진 텍스트 금지** | 한글/영문이 깨지거나 알아볼 수 없는 문자열로 렌더링되면 안 된다 |
| **박스/패널 완결성** | 모든 박스의 테두리, 헤더 영역, 본문 영역이 완전하게 렌더링되어야 한다. 박스가 반만 렌더링되거나 테두리가 누락되면 안 된다 |
| **격자 정렬** | 2×2, 2×3, 3×3 격자 배치에서 박스의 크기와 정렬이 균등해야 한다. 한쪽으로 치우치거나 크기가 불균형하면 안 된다 |
| **아이콘 선명도** | 플랫 아이콘(채워진 원 안 흰색 심볼)이 선명하고 식별 가능하게 렌더링되어야 한다 |
| **의미적 일관성** | 이미지에 렌더링된 모든 텍스트가 의미적으로 일관되어야 한다. 제목과 본문의 맥락이 맞아야 하며, 관련 없는 텍스트가 섞이면 안 된다 |

### 프롬프트 작성 시 완결성 확보 방법

| 기법 | 설명 |
|------|------|
| **구체적 시각 요소 기술** | `[아이콘]` 대신 "채워진 원 안에 흰색 기어 심볼" 처럼 구체적으로 기술 |
| **텍스트 항목 명시** | 렌더링할 텍스트를 CONTENT에 정확히 나열. 모호한 참조(`위 내용 참조`) 금지 |
| **격자 구조 명시** | INSTRUCTION에서 격자 배치(2×2, 상단+중앙+하단 등)를 구체적으로 기술 |
| **번호+텍스트 세트** | 번호 뱃지와 텍스트를 함께 기술하여 순서가 명확하게 렌더링되도록 한다 |

## 구성 지시어 렌더링 방지 (Content Placement 메타라벨 금지) — CRITICAL

INSTRUCTION 블록의 Content Placement에서 CONTENT 텍스트의 배치를 설명할 때, **역할 분류명(메타라벨)**을 사용하면 Gemini가 해당 라벨을 이미지에 텍스트로 렌더링합니다. 반드시 CONTENT에 기재된 **실제 텍스트를 직접 인용**하여 배치를 지시합니다.

| 금지 (메타라벨 — 렌더링 유출) | 올바른 (실제 텍스트 인용) |
|------|------|
| `박스 하단: 보조 지표 텍스트 소수 배치` | `하단 영역에 '단일 모델 운용'과 '확장성 고수준' 텍스트를 소형으로 나란히 배치` |
| `상단 배너: 핵심 비전 텍스트 배치` | `상단 배너에 '제조업 디지털 전환 선도' 텍스트를 대형 볼드로 배치` |
| `우측: 기능 설명 나열` | `우측에 '기종 무관 범용 적용', '실시간 환경 학습' 텍스트를 세로로 나열` |
| `하단: 주요 성과 라벨` | `하단에 '불량률 30% 감소', '생산성 25% 향상' 텍스트를 나란히 배치` |

**금지 메타라벨 목록**: 보조 지표, 보조 텍스트, 핵심 성과, 주요 모듈, 기능 설명, 핵심 비전, 핵심 메시지, KPI 영역, 비전 영역, 결론 영역, 성과 영역, 메인 타이틀, 핵심 모듈명, 입력 라벨, 출력 라벨, 적용 사례, 세부기술, 공통 요소

## 출력 구조 매핑 (Output Structure Mapping)

gov 테마의 프롬프트는 다음과 같이 4-block 구조에 매핑됩니다. 모든 테마가 동일한 4-block 구조를 따릅니다.

### INSTRUCTION 블록 포함 내용
- `### Image Purpose`: 정부/공공기관 슬라이드의 목적 (정책 설명, 연구 발표 등)
- `### Target Audience`: 평가위원, 정부 관계자 등
- `### Key Message`: 전달할 핵심 메시지 한 문장
- `### Visual Style`: `gov` 스타일 명시 + 공식적/전문적 특성
- `### Rendering Style`: 서피스(2px 실선 박스), 배경(연한 그레이+배너), 코너/엣지(완전 직각), 연결선(2px+채운 화살표), 시각 장식(플랫 아이콘, 번호 뱃지), 공간 구성(엣지-투-엣지 격자), 시각 메타포(플랫 인포그래픽) 7개 차원
- `### Content Placement`: CONTENT에 기재된 **실제 텍스트를 직접 인용**(작은따옴표)하여 영역별 배치 위치 설명 (배너, 격자 박스, 주석 영역 등). 시각 요소(아이콘, 연결선, 표) 배치. **메타라벨(역할 분류명) 사용 금지**

### CONFIGURATION 블록 포함 내용
- `### Canvas Settings`: 3840 x 2160, 배경색 (테마별)
- `### Background Treatment`: 단색 + 상단 배너 + 하단 구분선
- `### Color Palette`: 주조/보조/강조/배경 4색 (9종 무드별)
- `### Layout Structure`: Org-Network/Swimlane/Timeline/Structure 중 선택 + 영역 구분
- `### Typography`: 3단 위계(대/중/소), 좌측 정렬, 번호 매김

### CONTENT 블록 포함 내용
- `### Title Area`: 메인 제목 (번호 목록 형태)
- `### Main Content`: 이미지에 렌더링될 순수 텍스트만 번호 목록으로 나열 (역할 라벨, 영역 구분자, 배치 정보 절대 포함 금지)
- `### Data Elements`: 성과 지표, 수치 데이터

### FORBIDDEN ELEMENTS 블록 포함 내용
- pt/px 단위, 한영 병기, ASCII 힌트, 렌더링 지시문, 폰트 지정, 좌표, 플레이스홀더, 위치 지시자, 레이아웃 유형명, 색상 코드, 역할 라벨, 기관 로고/마크, 기관명/부처명, 국가 상징, 테이블 형식 CONTENT 등
