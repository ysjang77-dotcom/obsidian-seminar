# Pitch (피치덱) 테마

**용도**: 간결하고 임팩트 있는 피치덱 스타일의 시각자료를 생성합니다. 문제→해결→시장→팀 흐름으로 투자자/의사결정자를 설득합니다.

**시각 정체성**: "Apple 키노트처럼 — 어두운 그래디언트 위의 거대한 숫자와 프로스티드 글래스 카드"

**활용 상황**:
- 스타트업 투자 유치 발표
- 신규 프로젝트 제안서
- 경영진 대상 1-pager
- 데모데이/IR 발표 자료
- 핵심 성과 요약 슬라이드

## 색상 팔레트

| 역할 | 색상 | HEX | 용도 |
|------|------|-----|------|
| 주조 | 스노우 화이트 | #F5F5F7 | 제목, 핵심 메시지, 숫자 강조 (밝은 텍스트) |
| 보조 | 애플 그레이 | #86868B | 본문, 보조 텍스트, 구분선 |
| 강조 | 애플 퍼플 | #BF5AF2 | CTA, 핵심 수치, 하이라이트, 그래디언트 포인트 |
| 강조 보조 | 애플 블루 | #0A84FF | 보조 하이라이트, 그래디언트 보조, 스파크라인 |
| 배경 | 퓨어 블랙 | #000000 | 전체 배경 기반 (다크 그래디언트의 베이스) |

**팔레트 특징**:
- Apple 키노트/WWDC 프레젠테이션 스타일의 프리미엄 다크 테마
- 어두운 배경 위 밝은 텍스트로 극강의 대비와 집중력
- 퍼플-블루 그래디언트가 혁신과 프리미엄 이미지 동시 전달
- 어두운 배경이 여백을 더 깊고 드라마틱하게 만든다

## 권장 레이아웃

### Z-Pattern (최우선 권장)
- **(1) 좌상**: 핵심 문제 또는 제목
- **(2) 우상**: 임팩트 숫자 (시장 규모, 성장률 등)
- **(3) 좌하**: 솔루션 핵심 (한 문장)
- **(4) 우하**: CTA 또는 요청사항

```
(1) [문제/제목] ─────────────── (2) $10B+ 시장
        \
         \
(3) [솔루션 한 줄] ─────────────── (4) [CTA/요청]
```

**장점**: 30초 내 핵심 메시지 전달, 시선 흐름 최적화

### Single-Hero (임팩트 최대화)
- 화면 중앙에 **하나의 대형 숫자/메시지**만 배치
- 하단에 한 줄 부연 설명

```
┌─────────────────────────────┐
│                             │
│          +340%              │
│      월간 활성 사용자 증가     │
│                             │
└─────────────────────────────┘
```

## 스타일 노트

### 숫자 강조 규칙
- **크기**: 핵심 숫자는 화면의 30% 이상 차지
- **단위**: 숫자 옆에 단위 명시 ($, %, 배, 명 등)
- **맥락**: 숫자 아래 한 줄로 의미 설명

### 텍스트 원칙
- **한 슬라이드 한 메시지**: 복수 아이디어 배제
- **동사 사용**: "AI 기반 솔루션" → "AI로 해결합니다"
- **구체적 수치**: "크게 성장" → "3배 성장"

### 여백 활용
- **최소 30% 여백**: 어두운 여백이 메시지의 무게감을 증폭
- **밀집 배제**: 빽빽한 내용은 피치덱에 부적합

## 콘텐츠 표현 규칙

pitch 스타일의 텍스트는 **개조식 명사구**를 따릅니다:

| 규칙 | 설명 | 예시 |
|------|------|------|
| 개조식 | 핵심 키워드 중심의 간결한 명사구 | ✗ "고령 농부가 수동으로 트랙터를 운전한다" → ✓ "고령 농부의 수동 트랙터 운전" |
| 서술어 최소화 | 종결어미(~다, ~합니다) 대신 명사형 종결 | ✗ "시장이 빠르게 성장하고 있다" → ✓ "시장 고속 성장" |
| 구체적 수치 | 추상적 표현 대신 수치 사용 | ✗ "크게 성장" → ✓ "3배 성장" |
| 임팩트 명사구 | 대형 숫자 + 한 줄 명사구 조합 | "+340% 월간 활성 사용자 증가" |
| **한글 단독 표기** | 한영 병기 절대 금지. 한글 또는 영문 약어 하나만 사용 | ✗ "연구 (Research)", "분석(Analysis)" → ✓ "연구", "분석" / 약어만 허용: "AI", "IoT" |

## 렌더링 스타일 (Rendering Style)

이 스타일의 시각적 DNA를 정의합니다. prompt-designer가 INSTRUCTION/CONFIGURATION 블록 생성 시 반드시 참조해야 합니다.

| 차원 | 지시 사항 |
|------|-----------|
| **서피스 (박스/패널)** | 프로스티드 글래스(frosted glass) 카드: 반투명 배경(rgba(255,255,255,0.08)) + 1px rgba(255,255,255,0.15) 테두리 + 배경 블러(backdrop blur). 카드 내부 텍스트는 밝은 색(#F5F5F7). 카드가 어두운 배경 위에 은은하게 떠 있는 느낌. 불투명 흰색 카드 사용 금지 — 반드시 반투명 글래스 효과. |
| **배경** | **Apple 다크 그래디언트**: #000000(퓨어 블랙) 기반에 부드럽게 흐르는 컬러 워시. 좌상단에서 우하단으로 #1C1C4E(미드나이트 인디고) → #2D1B69(딥 퍼플) → #000000 대각선 그래디언트. 보조 블롭: #0A84FF(애플 블루) 15% opacity 원형 블롭 1개, #BF5AF2(애플 퍼플) 10% opacity 블롭 1개를 배경에 자연스럽게 배치. 그래디언트는 미묘하고 우아하게 — 네온이나 사이버펑크 느낌 금지. |
| **코너/엣지** | 중간 라운딩(16px radius). Apple 카드 스타일. 숫자 강조 블록은 pill shape(완전 라운드) 허용. 글래스 카드의 테두리는 얇고 은은하게(1px, 흰색 15% opacity). |
| **연결선** | 연결선 최소화. 쓸 경우 1px rgba(255,255,255,0.2) 점선. 숫자와 메시지가 자체로 연결을 나타낸다. 복잡한 다이어그램 자체를 지양한다. |
| **시각 장식** | 그래디언트 컬러 워시(유기적 블롭) 배경 장식 1-2개. 핵심 숫자 뒤에 강조색(#BF5AF2) 은은한 글로우 효과. 미니 스파크라인 차트(밝은 색). 장식은 절제되고 우아하게 — 과도한 효과 금지. |
| **타이포그래피** | 숫자가 가장 크다. 핵심 메트릭(+340%, $10B)이 화면의 30% 이상 차지하며 **#F5F5F7(스노우 화이트)** 또는 **#BF5AF2(퍼플 강조)** 색상. 제목보다 숫자가 크다. 제목은 #F5F5F7 세미볼드 중간 크기. 보조 텍스트는 #86868B. CTA는 강조색(#BF5AF2) 볼드. |
| **공간 구성** | 대담한 비대칭. 핵심 숫자가 좌측 또는 중앙 상단에 매우 크게. 나머지 요소들이 어두운 공간에 작게 흩어진다. Z-패턴 시선 유도. 30% 이상 의도적 어두운 여백. 어두운 배경의 깊이감이 콘텐츠를 더욱 부각시킨다. |
| **시각 메타포** | Apple 키노트 프레젠테이션(Apple Keynote Presentation). 어두운 무대 위의 스포트라이트처럼 핵심 메시지만 밝게 빛난다. 큰 숫자 카드, 미니 스파크라인, 프로스티드 글래스 패널. 프리미엄 테크 브랜드의 제품 발표 느낌. |

### 다크 테마 시인성 규칙 (Dark Theme Legibility) — CRITICAL

어두운 배경 위에서 텍스트와 카드의 **가독성을 보장**하기 위한 필수 규칙:

| 규칙 | 설명 |
|------|------|
| **텍스트 대비** | 모든 본문 텍스트는 어두운 배경과 **최소 4.5:1 대비비** 확보. 주조 텍스트(#F5F5F7)는 검정 배경 위에서 약 18:1 대비비로 충분. 보조 텍스트(#86868B)는 보조 정보에만 사용하고 핵심 정보에는 사용하지 않는다 |
| **글래스 카드 가독성** | 글래스 카드 내부 텍스트는 반드시 밝은 색(#F5F5F7 또는 #FFFFFF). 카드 배경의 반투명도가 텍스트 가독성을 해치면 opacity를 높이거나 텍스트에 미세한 드롭섀도우를 추가한다 |
| **숫자 강조** | 핵심 숫자는 #F5F5F7(스노우 화이트) 또는 #BF5AF2(퍼플 강조) 사용. 어두운 배경 위에서 즉각 눈에 들어오도록 충분히 크고 밝게 렌더링한다 |
| **그래디언트 위 텍스트** | 배경 그래디언트 블롭이나 컬러 워시 위에 텍스트가 놓이는 경우, 텍스트 뒤에 미세한 어두운 비네팅(vignette) 또는 반투명 오버레이를 두어 가독성을 확보한다 |
| **금지: 어두운 텍스트** | 어두운 배경 위에 어두운 색(#0A2540, #425466 등) 텍스트 사용 절대 금지. 모든 텍스트는 밝은 색이어야 한다 |

## 렌더링 금지 요소 (CRITICAL)

프롬프트 생성 시 다음 요소는 **CONTENT BLOCK에 절대 포함 금지**:

| 유형 | 금지 예시 | 설명 |
|------|-----------|------|
| **이미지 플레이스홀더** | `[Image 1]`, `[Image 2]`, `[사진]`, `[이미지]`, `[아이콘]` | **대괄호 이미지 참조는 Gemini가 그대로 텍스트로 렌더링함.** 시각 요소는 INSTRUCTION의 Content Placement에서 자연어로 묘사할 것 |
| 위치 지시자 | `[상단]`, `[하단 결론1]`, `[왼쪽 영역]` | 레이아웃 배치 힌트 |
| 레이아웃 유형명 | `Z-Pattern`, `Pitch Deck Layout` | 메타포/레이아웃 이름 |
| 색상 코드 | `(#HEXCODE)`, `Primary Color` | CONFIGURATION에서만 사용 |
| 크기 힌트 | `(대형)`, `(중형)`, `48pt` | INSTRUCTION에서만 사용 |
| 역할 라벨 | `Main Title`, `Sub-header` | INSTRUCTION에서만 사용 |
| **한영 병기 텍스트** | `연구 (Research)`, `분석(Analysis)`, `목표 / Goal` | 한글과 영어를 괄호·슬래시로 병기하는 모든 형태 금지. 한글 단독 또는 고유 영문 약어(AI, IoT, CNN 등) 단독만 허용 |

CONTENT에는 **순수한 텍스트 내용만** 포함합니다.

## 렌더링 완결성 (Rendering Completeness) — CRITICAL

이 테마의 Gemini 렌더링 결과물에서 다음 품질 기준을 반드시 충족해야 합니다:

| 규칙 | 설명 |
|------|------|
| **플레이스홀더 금지** | `[Image 1]`, `[Image 2]`, `[사진]`, `[아이콘]` 등 대괄호 플레이스홀더가 이미지에 렌더링되면 안 된다. 모든 시각 요소는 구체적으로 기술하여 Gemini가 실제로 렌더링하도록 한다 |
| **깨진 텍스트 금지** | 한글/영문이 깨지거나(garbled), 알아볼 수 없는 문자열로 렌더링되면 안 된다. 텍스트는 의미를 가진 완전한 단어/문장이어야 한다 |
| **프로스티드 글래스 텍스트 시인성** | 프로스티드 글래스 카드 위의 텍스트는 반드시 읽을 수 있어야 한다. 카드 배경의 반투명도가 텍스트 가독성을 해치면 opacity를 높이거나 텍스트에 미세한 드롭섀도우를 추가한다 |
| **숫자 렌더링 정확성** | 핵심 메트릭 숫자(+340%, $10B, 99.2% 등)가 정확하게 렌더링되어야 한다. 숫자가 잘리거나, 겹치거나, 읽을 수 없으면 안 된다 |
| **카드 완결성** | 글래스 카드의 테두리, 배경 블러, 내부 텍스트가 모두 완전하게 렌더링되어야 한다. 카드가 반만 렌더링되거나 테두리가 누락되면 안 된다 |
| **의미적 일관성** | 이미지에 렌더링된 모든 텍스트가 의미적으로 일관되어야 한다. 제목과 본문의 맥락이 맞아야 하며, 관련 없는 텍스트가 섞이면 안 된다 |

### 프롬프트 작성 시 완결성 확보 방법

| 기법 | 설명 |
|------|------|
| **구체적 시각 요소 기술** | `[Image]` 대신 "미니 스파크라인 차트(밝은 색 상승 곡선)" 처럼 구체적으로 기술 |
| **텍스트 항목 명시** | 렌더링할 텍스트를 CONTENT에 정확히 나열. 모호한 참조(`위 내용 참조`) 금지 |
| **카드 내용 완전 기술** | 글래스 카드 안에 들어갈 텍스트를 CONTENT에 빠짐없이 나열 |
| **숫자+단위+맥락 세트** | 숫자만 쓰지 않고 "월간 활성 사용자 +340%" 처럼 단위와 맥락을 함께 기술 |

## 구성 지시어 렌더링 방지 (Content Placement 메타라벨 금지) — CRITICAL

INSTRUCTION 블록의 Content Placement에서 CONTENT 텍스트의 배치를 설명할 때, **역할 분류명(메타라벨)**을 사용하면 Gemini가 해당 라벨을 이미지에 텍스트로 렌더링합니다. 반드시 CONTENT에 기재된 **실제 텍스트를 직접 인용**하여 배치를 지시합니다.

| 금지 (메타라벨 — 렌더링 유출) | 올바른 (실제 텍스트 인용) |
|------|------|
| `좌상: 핵심 문제 텍스트` | `좌상에 '제조 불량률 15%' 텍스트를 세미볼드로 배치` |
| `우상: 임팩트 숫자 (시장 규모)` | `우상에 '$10B+' 수치를 대형 볼드로 배치, 아래에 '글로벌 시장 규모' 소형 텍스트` |
| `우하: CTA 텍스트` | `우하에 '시리즈 A 50억원 모집' 텍스트를 강조색으로 배치` |

**금지 메타라벨 목록**: 보조 지표, 보조 텍스트, 핵심 성과, 주요 모듈, 기능 설명, 핵심 비전, 핵심 메시지, KPI 영역, 비전 영역, 결론 영역, 성과 영역, 메인 타이틀, CTA 영역, 임팩트 숫자

## 출력 구조 매핑 (Output Structure Mapping)

pitch 테마의 프롬프트는 다음과 같이 4-block 구조에 매핑됩니다. 모든 테마가 동일한 4-block 구조를 따릅니다.

### INSTRUCTION 블록 포함 내용
- `### Image Purpose`: 피치덱 슬라이드의 목적 (투자 유치, 성과 요약 등)
- `### Target Audience`: 투자자, VC, 경영진 등
- `### Key Message`: 전달할 핵심 메시지 한 문장
- `### Visual Style`: `pitch` 스타일 명시 + Apple 키노트 특성
- `### Rendering Style`: 서피스(프로스티드 글래스), 배경(Apple 다크 그래디언트), 코너/엣지(16px 라운딩), 연결선(최소화), 시각 장식(그래디언트 블롭, 글로우), 공간 구성(대담한 비대칭), 시각 메타포(Apple 키노트) 7개 차원
- `### Content Placement`: CONTENT에 기재된 **실제 텍스트를 직접 인용**(작은따옴표)하여 영역별 배치 위치 설명 (Z-Pattern 또는 Single-Hero 배치). 시각 요소(스파크라인, 글로우 효과) 배치. **메타라벨(역할 분류명) 사용 금지**

### CONFIGURATION 블록 포함 내용
- `### Canvas Settings`: 3840 x 2160, 배경색 #000000
- `### Background Treatment`: Apple 다크 그래디언트 상세 (미드나이트 인디고 → 딥 퍼플 → 퓨어 블랙), 보조 블롭
- `### Color Palette`: 주조(#F5F5F7), 보조(#86868B), 강조(#BF5AF2), 강조 보조(#0A84FF), 배경(#000000)
- `### Layout Structure`: Z-Pattern 또는 Single-Hero + 영역 구분
- `### Typography`: 숫자가 가장 크다 규칙, 제목/본문/CTA 위계

### CONTENT 블록 포함 내용
- `### Title Area`: 핵심 제목/선언문 (번호 목록 형태)
- `### Main Content`: 이미지에 렌더링될 순수 텍스트만 번호 목록으로 나열 (역할 라벨, 영역 구분자, 배치 정보 절대 포함 금지)
- `### Data Elements`: 핵심 수치/KPI (숫자+단위+맥락 세트)

### FORBIDDEN ELEMENTS 블록 포함 내용
- pt/px 단위, 한영 병기, ASCII 레이아웃 힌트, 렌더링 지시문, 폰트 지정, 좌표, 플레이스홀더, 위치 지시자, 레이아웃 유형명, 색상 코드, 역할 라벨, 테이블 형식 CONTENT 등
