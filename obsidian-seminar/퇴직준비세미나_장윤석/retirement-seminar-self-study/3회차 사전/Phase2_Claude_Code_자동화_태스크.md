## 1. AI Agent 개념 이해

### 1.1 AI Agent란?

AI Agent는 단순히 질문에 답하는 것을 넘어, **도구를 사용하고, 계획을 세우고, 실행하는** 자율적인 AI 시스템입니다.

```
┌─────────────────────────────────────────────────────────────┐
│  AI Agent의 3대 핵심 요소                                    │
├─────────────────────────────────────────────────────────────┤
│  1. Tool Use (도구 사용)                                     │
│     - 파일 읽기/쓰기, 웹 검색, 코드 실행 등                   │
│                                                             │
│  2. Planning (계획 수립)                                     │
│     - 복잡한 작업을 단계별로 분해                             │
│     - 최적의 실행 순서 결정                                   │
│                                                             │
│  3. Memory (기억 유지)                                       │
│     - 대화 맥락 기억                                         │
│     - 과거 작업 결과 참조                                     │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 Agent Loop 이해

```
    ┌──────────────┐
    │   사용자     │
    │   요청       │
    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │   계획       │  ← Agent가 작업 분해
    │   (Plan)     │
    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │   실행       │  ← 도구 사용하여 작업 수행
    │   (Execute)  │
    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │   검토       │  ← 결과 확인 및 판단
    │   (Review)   │
    └──────┬───────┘
           │
           ├─── 완료 ──→ 결과 전달
           │
           └─── 추가 작업 필요 ──→ 계획으로 돌아감
```

### 2.1 Claude Code 설치

- Node.js 18 이상 설치
- Claude 계정 (Pro 또는 Max 권장)

```bash

claude --version

# 3. 로그인 (브라우저 열림)

## 3. 반복 업무 선정 및 Agent화 설계

### 3.1 자동화 대상 업무 선정

다음 중 하나 또는 유사한 반복 업무를 선정하세요:

| 업무 유형 | 예시 |
|----------|------|
| 📊 데이터 정리 | Excel 데이터 정리, CSV 변환, 중복 제거 |
| 📝 문서 생성 | 일일 리포트, 회의록 초안, 보고서 템플릿 |
| 📧 이메일 작성 | 정기 공지, 협조 요청, 결과 공유 메일 |
| 📁 파일 관리 | 파일명 일괄 변경, 폴더 정리, 백업 |
| 🔍 정보 수집 | 웹에서 정보 검색 및 요약, 논문 초록 정리 |
| 📋 체크리스트 | 일일 점검 항목 생성, 진행 상황 체크 |

### 3.2 업무 분석 체크리스트

선정한 업무에 대해 다음을 분석하세요:

```markdown
## 업무 분석

### 1. 업무명
[예: 주간 업무 보고서 작성]

### 2. 현재 수행 방식
- 소요 시간: [예: 매주 금요일 30분]
- 반복 주기: [예: 매주 1회]
- 수작업 내용: [예: 이번 주 업무 정리, 다음 주 계획 작성]

### 3. 입력 정보
- [예: 이번 주 완료한 업무 목록]
- [예: 다음 주 예정 업무]
- [예: 특이사항/이슈]

### 4. 출력 결과물
- [예: 주간업무보고서.md 파일]
- [예: 형식: 마크다운 표 형식]

### 5. 자동화 가능 부분
- [예: 보고서 템플릿 생성]
- [예: 형식 통일 및 정리]

### 6. 사람이 해야 할 부분
- [예: 업무 내용 입력]
- [예: 최종 검토 및 수정]
```

### 3.3 Agent화 설계 문서 작성

```markdown
# Agent화 설계 문서

## 1. 프로젝트 개요

### 프로젝트명
[예: 주간업무보고서 자동 생성 Agent]

### 목표
[예: 입력된 업무 정보를 바탕으로 형식화된 주간업무보고서 자동 생성]

### 예상 효과
- 기존 소요 시간: 30분
- 자동화 후 예상 시간: 10분
- 절감 시간: 20분/주

---

## 2. Agent 설계

### 입력 (Input)
```
- 이번 주 완료 업무 (텍스트)
- 다음 주 예정 업무 (텍스트)
- 특이사항 (선택)
```

### 처리 과정 (Process)
```
1. 입력 정보 파싱
2. 보고서 템플릿 적용
3. 형식 정리 및 표 생성
4. 파일 저장
```

### 출력 (Output)
```
- 파일명: 주간업무보고서_YYYYMMDD.md
- 형식: 마크다운
```

[출력 형식]
마크다운 형식, 표 사용, A4 1페이지 분량

## 4. Agent 실행 및 결과물 획득

### 4.1 실행 예시

#### 예시 1: 파일 정리 Agent

```bash
# Claude Code 실행
> 현재 폴더의 모든 .txt 파일을 찾아서 
> 파일명에 오늘 날짜를 추가해주세요

# Agent 응답 예시:
# 1. 현재 폴더 스캔 중...
# 2. 발견된 .txt 파일: report.txt, notes.txt, memo.txt
# 3. 파일명 변경 계획:
#    - report.txt → report_20260120.txt
#    - notes.txt → notes_20260120.txt
#    - memo.txt → memo_20260120.txt
# 4. 실행하시겠습니까? (y/n)
```

#### 예시 2: 일일 리포트 생성 Agent

```bash
> 오늘 수행한 업무 내용을 입력받아서
> 일일업무보고서를 마크다운 형식으로 작성하고
> daily_report_오늘날짜.md 파일로 저장해주세요

# Agent가 대화형으로 정보 수집 후 파일 생성
```

#### 예시 3: 데이터 정리 Agent

```bash
> data.csv 파일을 읽어서
> 중복된 행을 제거하고
> 이름 열 기준으로 정렬해서
> data_cleaned.csv로 저장해주세요

# Agent가 파일 분석 후 자동 처리
```

### 4.2 Agent Loop 로그 캡처

Agent가 작업하는 각 단계를 캡처하세요:

| 단계 | 캡처 내용 |
|------|----------|
| 1. 요청 (Request) | 사용자가 입력한 명령어 |
| 2. 계획 (Plan) | Agent가 세운 실행 계획 |
| 3. 실행 (Execute) | 도구 사용 및 작업 수행 |
| 4. 결과 (Result) | 최종 결과물 |

## 5. 실습 시나리오: 일일 업무 리포트 Agent

### 5.1 시나리오 설명

매일 퇴근 전 작성하는 일일 업무 리포트를 Agent가 자동으로 생성합니다.

### 5.2 Step-by-Step 실행

#### Step 1: 프로젝트 폴더 생성

```bash
mkdir daily-report-agent
cd daily-report-agent
```

#### Step 2: Claude Code 실행

```bash
claude
```

#### Step 3: Agent에게 지시

```
당신은 일일 업무 리포트 생성을 도와주는 Agent입니다.

다음 절차를 수행해주세요:

1. 먼저 저에게 오늘 수행한 업무를 물어봐주세요
2. 내일 예정된 업무도 물어봐주세요
3. 특이사항이나 이슈가 있는지 확인해주세요
4. 입력받은 내용으로 일일업무보고서를 작성해주세요
5. 결과를 daily_report_[오늘날짜].md 파일로 저장해주세요

보고서 형식:
- 제목: 일일업무보고서 (날짜)
- 작성자: [나중에 입력]
- 금일 업무 내용 (표 형식)
- 명일 예정 업무
- 특이사항
```

#### Step 4: Agent와 대화하며 정보 제공

```
Agent: 오늘 수행한 업무를 알려주세요.
사용자: 
- AI 세미나 자료 준비
- 연구 데이터 분석
- 팀 회의 참석

Agent: 내일 예정된 업무를 알려주세요.
사용자:
- 세미나 발표
- 보고서 작성

Agent: 특이사항이나 이슈가 있나요?
사용자: 특별한 이슈 없음

Agent: 일일업무보고서를 작성하고 저장하겠습니다.
[파일 생성 중...]
완료: daily_report_20260120.md 파일이 생성되었습니다.
```

#### Step 5: 결과 확인

생성된 `daily_report_20260120.md`:

```markdown
# 일일업무보고서

**작성일**: 2026-01-20  
**작성자**: [이름]

---

## 금일 업무 내용

| 번호 | 업무 내용 | 상태 |
|------|----------|------|
| 1 | AI 세미나 자료 준비 | 완료 |
| 2 | 연구 데이터 분석 | 완료 |
| 3 | 팀 회의 참석 | 완료 |

---

## 명일 예정 업무

1. 세미나 발표
2. 보고서 작성

---

## 특이사항

- 특별한 이슈 없음

---

*이 보고서는 AI Agent에 의해 자동 생성되었습니다.*
```

## ❓ 자주 묻는 질문 (FAQ)

### Q1: Agent가 파일을 잘못 수정하면 어떻게 하나요?
**A**: Agent는 실행 전 확인을 요청합니다. "실행하시겠습니까?"라는 질문에 신중하게 답변하세요. 만약을 대비해 **Git으로 버전 관리**를 하거나, 작업 전 백업을 권장합니다.

### Q2: 자동화할 업무를 못 찾겠어요.
**A**: 다음 질문을 자문해보세요:
- 매일/매주 반복하는 작업이 있는가?
- 형식이 정해진 문서를 자주 작성하는가?
- 단순하지만 시간이 걸리는 작업이 있는가?

### Q3: Agent가 예상과 다르게 동작해요.
**A**: 프롬프트를 더 구체적으로 수정하세요. 특히:
- 단계별 지시를 명확히
- 예상 출력 형식을 구체적으로
- "~하지 마세요" 같은 제약 조건 추가
