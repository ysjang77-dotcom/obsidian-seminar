## Part 1: 프롬프트 골든 공식 (40분)

### 1.1 프롬프트란 무엇인가? (5분)

**프롬프트(Prompt)**는 AI에게 전달하는 **입력 텍스트**입니다.

```
사람 ──[프롬프트]──> AI ──[응답]──> 결과
```

>[!info] **핵심 인사이트**
> AI의 출력 품질은 **프롬프트의 품질**에 정비례합니다.
> 좋은 프롬프트 = 좋은 결과, 나쁜 프롬프트 = 나쁜 결과

---

### 1.2 프롬프트 골든 공식: RCIF (20분)

효과적인 프롬프트는 **4가지 요소**로 구성됩니다:

```
┌─────────────────────────────────────────────────┐
│           프롬프트 골든 공식 (RCIF)              │
├─────────────────────────────────────────────────┤
│  R: Role (역할)     - AI에게 역할을 부여한다     │
│  C: Context (맥락)  - 배경 정보를 제공한다       │
│  I: Instruction (지시) - 명확한 작업을 요청한다  │
│  F: Format (형식)   - 출력 형식을 지정한다       │
└─────────────────────────────────────────────────┘
```

#### 1️⃣ Role (역할) - "당신은 ~입니다"

AI에게 전문가 페르소나를 부여합니다.

| 역할 예시 | 효과 |
|-----------|------|
| "당신은 10년 경력의 기술 문서 작성자입니다" | 전문적인 문서 스타일 |
| "당신은 친절한 고객 서비스 담당자입니다" | 공손하고 배려하는 톤 |
| "당신은 엄격한 코드 리뷰어입니다" | 비판적 분석 |
| "당신은 연구원을 돕는 연구 조교입니다" | 학술적 관점 |

```markdown
✅ 좋은 예시:
"당신은 한국기계연구원에서 10년간 근무한 선임연구원입니다.
 연구 제안서 작성에 능숙하고, 정부 R&D 과제 구조를 잘 알고 있습니다."
```

#### 2️⃣ Context (맥락) - "상황은 이렇습니다"

AI가 작업을 이해하는 데 필요한 배경 정보를 제공합니다.

| 맥락 유형 | 예시 |
|-----------|------|
| 목적 | "이 문서는 팀 내 신입 연구원 교육용입니다" |
| 대상 독자 | "독자는 프로그래밍 경험이 없는 연구원입니다" |
| 제약 조건 | "분량은 A4 2페이지 이내입니다" |
| 이전 작업 | "지난 회의에서 이런 피드백이 있었습니다: ..." |

```markdown
✅ 좋은 예시:
"나는 지금 산업부 R&D 과제 중간보고서를 작성하고 있습니다.
 평가위원은 해당 분야 전문가 5인이며,
 보고서 분량은 30페이지 이내입니다.
 핵심 성과는 시뮬레이션 정확도 15% 향상입니다."
```

#### 3️⃣ Instruction (지시) - "이것을 해주세요"

AI가 수행해야 할 **구체적인 작업**을 명시합니다.

| 나쁜 지시 | 좋은 지시 |
|-----------|-----------|
| "이메일 써줘" | "김 부장님께 회의 일정 변경 요청 이메일을 작성해줘" |
| "요약해줘" | "이 논문의 핵심 기여 3가지를 각각 2문장으로 요약해줘" |
| "분석해줘" | "이 데이터에서 상위 5개 트렌드를 찾고 각각의 원인을 설명해줘" |

>[!tip] **지시 작성 팁**
> - 동사로 시작하세요: "작성해줘", "분석해줘", "비교해줘"
> - 수량을 명시하세요: "3가지", "5개 항목", "2문장으로"
> - 단계를 나누세요: "먼저 ~하고, 그 다음 ~해줘"

#### 4️⃣ Format (형식) - "이런 형태로 주세요"

AI의 출력 형식을 지정합니다.

| 형식 유형 | 지정 방법 |
|-----------|-----------|
| 글머리 기호 | "bullet point로 정리해줘" |
| 번호 목록 | "1, 2, 3 순서로 나열해줘" |
| 표 | "표 형식으로 정리해줘" |
| 마크다운 | "마크다운 형식으로 작성해줘" |
| JSON | "JSON 형식으로 출력해줘" |
| 특정 구조 | "제목/본문/결론 구조로 작성해줘" |

```markdown
✅ 좋은 예시:
"결과는 다음 형식의 표로 정리해줘:
| 항목 | 현재 상태 | 개선 방안 | 우선순위 |"
```

### 1.3 골든 공식 적용 예시 (15분)

#### 예시 1: 연구 제안서 초안 요청

```markdown
❌ 나쁜 프롬프트:
"연구 제안서 써줘"

✅ 좋은 프롬프트 (RCIF 적용):

[R: 역할]
당신은 정부 R&D 과제 제안서 작성에 20년 경력이 있는 연구기획 전문가입니다.

[C: 맥락]
나는 로봇 분야 선임연구원이며, 산업부 로봇산업핵심기술개발사업에 
신규 과제를 제안하려고 합니다. 과제명은 "협동로봇 안전제어 시스템 개발"이고,
총 연구기간 3년, 연구비 30억 규모입니다.

[I: 지시]
과제 제안서의 "연구 필요성" 섹션을 작성해주세요.
다음 내용을 포함해야 합니다:
1. 국내외 기술 동향 (최근 3년)
2. 현재 기술의 한계점
3. 본 연구의 차별성

[F: 형식]
마크다운 형식으로 작성하고, 각 섹션은 H3(###) 제목을 사용해주세요.
분량은 A4 2페이지(약 1500자) 내외입니다.
```

#### 예시 2: 회의록 정리 요청

```markdown
❌ 나쁜 프롬프트:
"회의 내용 정리해줘"

✅ 좋은 프롬프트 (RCIF 적용):

[R: 역할]
당신은 기업의 경영지원팀에서 10년간 회의록을 작성해온 전문 비서입니다.

[C: 맥락]
아래는 오늘 진행된 "프로젝트 킥오프 미팅" 녹취록입니다.
참석자: 김철수(PM), 이영희(개발), 박민수(디자인), 최지영(QA)

[회의 내용]
(녹취록 또는 메모 붙여넣기)

[I: 지시]
이 회의 내용을 공식 회의록으로 정리해주세요.
다음 항목을 반드시 포함해야 합니다:
1. 주요 논의 사항 (3-5개)
2. 결정 사항
3. 액션 아이템 (담당자, 마감일 포함)
4. 다음 회의 일정

[F: 형식]
다음 템플릿을 사용해주세요:
## 회의 정보
- 일시:
- 참석자:
- 주제:

## 주요 논의 사항
1. ...

## 결정 사항
- ...

## 액션 아이템
| 담당자 | 업무 | 마감일 |
```

## Part 3: Hallucination 이해와 검증 (40분)

### 3.1 Hallucination이란? (10분)

**Hallucination(환각)**은 AI가 **사실이 아닌 정보를 사실인 것처럼 생성**하는 현상입니다.

```
┌─────────────────────────────────────────────────┐
│  ⚠️ Hallucination 예시                          │
├─────────────────────────────────────────────────┤
│  Q: "김영삼 대통령의 노벨상 수상 연도는?"        │
│  A: "김영삼 대통령은 1998년 노벨 평화상을 수상..." │
│                                                  │
│  → 실제로 김영삼 대통령은 노벨상을 수상한 적 없음! │
└─────────────────────────────────────────────────┘
```

#### Hallucination이 발생하는 이유

1. **확률 기반 생성**: AI는 "가장 그럴듯한 다음 단어"를 예측
2. **학습 데이터 한계**: 잘못된 정보로 학습했을 수 있음
3. **지식 컷오프**: 학습 이후의 정보를 모름
4. **자신감 과잉**: "모른다"고 말하기보다 추측으로 답변

>[!warning] **핵심 경고**
> AI의 답변은 항상 **"그럴듯한 거짓"**일 수 있습니다.
> **검증 없이 AI 답변을 그대로 사용하면 위험합니다!**

### 3.2 Hallucination 라이브 데모 (10분)

#### 데모 1: 존재하지 않는 논문 인용

```
프롬프트: "협동로봇 안전에 관한 최신 논문 5편을 인용 형식으로 알려줘"
```

AI가 제시한 논문을 Google Scholar에서 검색해보세요. 
→ 존재하지 않는 논문일 가능성이 높습니다!

#### 데모 2: 가짜 통계

```
프롬프트: "한국의 협동로봇 시장 규모는 2025년 기준 얼마인가?"
```

AI가 제시한 수치의 출처를 확인해보세요.
→ 출처가 없거나 검증 불가능한 경우가 많습니다!

### 3.3 검증 체계 구축 (10분)

#### 3단계 검증 프로세스

```
┌─────────────────────────────────────────────────┐
│            Hallucination 검증 3단계             │
├─────────────────────────────────────────────────┤
│ Step 1: 출처 요청                               │
│   "위 내용의 출처를 알려줘"                     │
│   → 출처가 없으면 의심!                         │
├─────────────────────────────────────────────────┤
│ Step 2: 교차 검증                               │
│   Google, 논문 DB, 공식 사이트에서 확인          │
│   → 2개 이상의 독립적 출처로 검증               │
├─────────────────────────────────────────────────┤
│ Step 3: 전문가 검토                             │
│   해당 분야 전문가에게 확인                      │
│   → 중요한 내용은 반드시 전문가 검토            │
└─────────────────────────────────────────────────┘
```

#### 검증 프롬프트 템플릿

```markdown
검증 요청 프롬프트:

"방금 답변한 내용 중에서:
1. 확실히 사실인 것
2. 불확실한 것
3. 추측인 것
을 구분해서 알려줘. 
각 항목에 대해 신뢰도를 상/중/하로 표시해줘."
```

## Part 4: 업무 프롬프트 워크숍 (30분)

### 4.1 실전 팁: 프롬프트 개선 기법 (10분)

#### 기법 1: 예시 제공 (Few-shot)

```markdown
"다음 예시처럼 작성해줘:

입력: 회의 일정 변경 요청
출력: 
안녕하세요 김 팀장님,
다음 주 월요일 예정된 회의 일정 변경을 요청드립니다.
[변경 사유]
[대안 제시]
감사합니다.

이제 내 요청을 처리해줘:
입력: 휴가 신청
출력: "
```

#### 기법 2: 단계별 사고 (Chain of Thought)

```markdown
"이 문제를 해결해줘. 단, 다음 단계로 진행해줘:
1. 먼저 문제를 분석해줘
2. 가능한 해결책 3가지를 나열해줘
3. 각 해결책의 장단점을 비교해줘
4. 최선의 해결책을 추천하고 이유를 설명해줘"
```

#### 기법 3: 반복 개선 (Iterative Refinement)

```markdown
1차: "연구 제안서 초안 작성해줘"
   → 결과 확인

2차: "좋아. 그런데 '연구 필요성' 부분을 더 구체적인 
     데이터와 함께 보강해줘"
   → 결과 확인

3차: "마지막으로, 전체 문서의 톤을 좀 더 
     학술적으로 다듬어줘"
   → 최종 결과
```

>[!tip] **반복 개선의 핵심**
> 한 번에 완벽한 결과를 기대하지 마세요.
> 작은 개선을 반복하면 더 좋은 결과를 얻습니다.

## 📝 오늘 배운 내용 정리

### 핵심 요약

1. **프롬프트 골든 공식 (RCIF)**
   - Role (역할): AI에게 전문가 페르소나 부여
   - Context (맥락): 배경 정보와 제약 조건 제공
   - Instruction (지시): 구체적인 작업 요청
   - Format (형식): 출력 형식 지정

2. **Good vs Bad 프롬프트**
   - 나쁜 프롬프트: 모호하고 일반적
   - 좋은 프롬프트: 구체적이고 맥락이 풍부함

3. **Hallucination 검증**
   - AI는 "그럴듯한 거짓"을 생성할 수 있음
   - 3단계 검증: 출처 요청 → 교차 검증 → 전문가 검토

4. **실전 기법**
   - 예시 제공 (Few-shot)
   - 단계별 사고 (Chain of Thought)
   - 반복 개선 (Iterative Refinement)
