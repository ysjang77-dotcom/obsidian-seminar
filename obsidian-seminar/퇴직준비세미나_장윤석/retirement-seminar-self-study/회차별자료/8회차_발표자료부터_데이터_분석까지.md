## Part 1: AI 기반 프레젠테이션 제작 워크플로우 (40분)

### 1.1 전통적 방식 vs AI 워크플로우 (10분)

전통적인 발표자료 제작은 슬라이드 디자인에 가장 많은 시간을 소비합니다. AI 워크플로우는 **구조 설계(Outline)**에 집중하고 디자인은 AI에게 맡깁니다.

#### 1.1.1 패러다임의 변화
- **과거의 방식**:
    1. 아이디어 구상 (1시간)
    2. 슬라이드 하나씩 제작 (5시간)
    3. 디자인 수정 및 폰트 맞추기 (2시간)
    4. 내용 보강 및 오타 수정 (1시간)
    - 총 소요 시간: 약 9시간
- **AI 워크플로우**:
    1. 아이디어 구상 및 Claude와 구조 설계 (30분)
    2. AI 디자인 도구로 슬라이드 자동 생성 (5분)
    3. 세부 내용 조정 및 데이터 업데이트 (1시간)
    4. 최종 검토 (25분)
    - 총 소요 시간: 약 2시간

#### 1.1.2 AI 워크플로우의 핵심 가치
1. **속도(Speed)**: 며칠 걸리던 초안 작성이 10분 내외로 단축됩니다.
2. **일관성(Consistency)**: 전체 슬라이드의 톤앤매너와 디자인 통일성이 자동으로 유지됩니다.
3. **창의성(Creativity)**: 내가 생각지 못한 시각화 방식이나 구조를 AI가 제안합니다.
4. **집중(Focus)**: 단순 반복 작업에서 벗어나 연구의 본질적인 메시지에 집중할 수 있습니다.

---

### 1.2 슬라이드 구조 설계 (15분)

좋은 발표자료의 시작은 논리적인 구조입니다. Claude를 활용하여 청중의 눈높이에 맞는 목차와 슬라이드별 핵심 메시지를 도출합니다.

#### 1.2.1 단계별 프롬프트 전략 (RCIF 적용)
1. **Role (역할)**: "당신은 20년 경력의 기술 프레젠테이션 전문가이자 로봇 공학 박사입니다."
2. **Context (맥락)**: "나는 다음 주 국제 학회에서 '협동로봇 안전 제어 알고리즘'에 대해 발표해야 합니다. 청중은 해당 분야 전문가와 대학원생들입니다."
3. **Instruction (지시)**: "총 15분 분량(슬라이드 12장)의 발표 구조를 짜주세요. 각 슬라이드마다 제목, 핵심 메시지 3가지, 그리고 추천하는 시각 자료(차트, 다이어그램 등)를 포함해야 합니다."
4. **Format (형식)**: "마크다운 표 형식으로 정리해주세요."

#### 1.2.2 실전 프롬프트 예시
> "당신은 한국기계연구원의 수석연구원입니다. 이번에 개발한 'AI 기반 예지보전 시스템'의 성과를 경영진에게 보고하기 위한 발표자료 구조를 만들어주세요. 
> 
> **제약 조건**:
> - 슬라이드 분량: 8장 이내
> - 핵심 타겟: 기술적 세부사항보다는 경제적 효과와 도입 타당성에 관심이 많은 경영진
> - 포함 내용: 기존 방식의 문제점, AI 도입 후 효율성 향상 지표, 향후 로드맵
> 
> 각 슬라이드별로 스크립트 초안도 함께 작성해주세요."

### 1.3 AI 디자인 도구 활용 (15분)

생성된 텍스트 구조를 바탕으로 슬라이드를 자동 생성하는 도구들을 소개합니다.

#### 1.3.1 주요 도구 상세 비교
| 도구명 | 주요 특징 | 장점 | 단점 |
|--------|-----------|------|------|
| **Gamma.app** | 텍스트 입력 시 전체 디자인 자동 생성 | 압도적인 속도, 세련된 템플릿 | 세부 커스터마이징의 한계 |
| **Beautiful.ai** | 데이터 기반 스마트 템플릿 | 차트와 레이아웃 자동 최적화 | 유료 플랜 위주의 기능 |
| **Canva Magic** | 키워드 기반 디자인 생성 | 방대한 이미지 리소스 활용 가능 | 구조적 논리성보다는 비주얼 중심 |
| **Claude Artifacts** | React 기반 인터랙티브 슬라이드 | 시뮬레이션이나 코드 데모에 최적 | 마크다운/코드 지식 필요 |

#### 1.3.2 Gamma.app 실습 가이드 (Step-by-Step)
1. **접속**: [Gamma.app](https://gamma.app/)에 접속하여 구글 계정으로 로그인합니다.
2. **생성 방식 선택**: 'Create new with AI' -> 'Text to Deck'을 선택합니다.
3. **내용 입력**: Claude에서 생성한 슬라이드 구조와 내용을 복사하여 입력창에 붙여넣습니다.
4. **설정 조정**: 언어를 'Korean'으로 설정하고, 이미지 스타일(Professional, Minimal 등)을 선택합니다.
5. **테마 선택**: AI가 제안하는 여러 테마 중 발표 성격에 맞는 것을 고릅니다.
6. **생성 및 수정**: 'Generate' 버튼을 누르면 실시간으로 슬라이드가 만들어집니다. 이후 텍스트를 수정하거나 이미지를 교체합니다.

## Part 2: Claude for Excel 데이터 분석 자동화 (40분)

### 2.1 엑셀의 한계를 넘는 AI (10분)

복잡한 VLOOKUP, INDEX/MATCH, 또는 VBA 매크로 작성이 필요할 때 Claude는 강력한 조력자가 됩니다.

#### 2.1.1 AI가 해결해주는 엑셀의 고질적 문제
- **수식 오류 해결**: "이 수식에서 #VALUE! 에러가 왜 나는지 알려주고 수정해줘."
- **데이터 정제 자동화**: "A열에 섞여 있는 '2026.01.28', '26/01/28', 'Jan 28, 2026' 형식을 모두 'YYYY-MM-DD'로 통일하는 수식을 알려줘."
- **복잡한 조건 처리**: "B열이 '완료'이고 C열의 날짜가 오늘 이전인 행들만 찾아서 D열의 합계를 구하는 수식을 작성해줘."

#### 2.1.2 자연어 → 수식 변환 실습
- **요청**: "A열에는 제품명, B열에는 판매량, C열에는 단가가 있어. 판매량이 100개 이상인 제품들에 대해서만 '단가 * 판매량 * 0.9'를 계산하고, 나머지는 '단가 * 판매량'을 계산하는 수식을 만들어줘."
- **결과**: `=IF(B2>=100, C2*B2*0.9, C2*B2)`

---

### 2.2 실습: 데이터 분석 파이프라인 (30분)

제공된 연구 데이터(CSV)를 활용하여 다음 작업을 수행합니다.

#### 2.2.1 데이터 구조 파악 및 전처리
1. Claude에게 데이터의 헤더(Header)와 샘플 데이터 5행을 텍스트로 복사하여 제공합니다.
2. **프롬프트**: "이 데이터는 센서 측정 기록입니다. 'Timestamp' 열을 기준으로 데이터를 정렬하고, 'Value' 열에 있는 이상치(Outlier, 평균에서 3표준편차 이상 벗어난 값)를 찾는 방법을 알려줘."

#### 2.2.2 VBA 매크로를 활용한 자동화
- **상황**: 매일 같은 형식의 데이터 파일을 받아서 특정 리포트를 만들어야 함.
- **요청**: "매번 시트의 A열부터 E열까지 데이터를 읽어서, E열의 값을 기준으로 피벗 테이블을 만들고 차트를 생성하는 VBA 코드를 작성해줘. 버튼 하나로 실행되게 하고 싶어."
- **적용**: Claude가 준 코드를 엑셀의 [개발 도구] -> [Visual Basic]에 붙여넣고 실행합니다.

#### 2.2.3 데이터 인사이트 도출
- 분석된 통계 수치를 Claude에게 보여주며 질문합니다.
- "실험군 A와 B의 평균값 차이가 15% 발생했어. 이 결과가 통계적으로 유의미한지 판단하기 위해 어떤 테스트를 해야 하고, 엑셀에서 어떻게 수행하는지 알려줘."

## Part 3: nanobanana-pro 인터랙티브 대시보드 (40분)

### 3.1 코딩 없는 대시보드 제작 (10분)

**nanobanana-pro**는 데이터를 업로드하고 원하는 시각화를 말로 설명하면 즉시 웹 기반 대시보드를 만들어주는 도구입니다.

#### 3.1.1 왜 nanobanana인가? (연구자 관점)
- **학습 곡선 제로**: Python의 Matplotlib이나 R의 ggplot2를 배우지 않아도 됩니다.
- **인터랙티브 요소**: 마우스 오버 시 상세 수치 표시, 특정 구간 확대 등이 기본 제공됩니다.
- **빠른 공유**: 연구 미팅 중에 즉석에서 데이터를 시각화하여 보여줄 수 있습니다.

#### 3.1.2 주요 시각화 기능
- **Dynamic Filtering**: 슬라이더를 움직여 특정 시간대의 데이터만 보기
- **Multi-Axis Charts**: 서로 다른 단위를 가진 두 데이터를 하나의 차트에 표시
- **Heatmaps & Maps**: 위치 데이터나 밀도 데이터를 직관적으로 표현

---

### 3.2 실습: 나만의 연구 대시보드 만들기 (30분)

#### 3.2.1 데이터 업로드 및 구조 확인
1. [nanobanana-pro](https://nanobanana-pro.co/) 사이트에 접속합니다.
2. 준비된 연구 데이터(CSV)를 드래그 앤 드롭으로 업로드합니다.
3. 각 컬럼의 데이터 타입(숫자, 날짜, 텍스트)이 올바르게 인식되었는지 확인합니다.

#### 3.2.2 시각화 요청 (Natural Language Query)
- **입력**: "X축은 'Date', Y축은 'Pressure'로 꺾은선 그래프를 그려줘. 선의 색상은 파란색으로 하고, 데이터 포인트마다 점을 찍어줘."
- **추가 요청**: "여기에 'Temperature' 데이터를 보조축(Secondary Axis)으로 추가하고 막대 그래프로 표시해줘."

#### 3.2.3 대시보드 구성 및 디자인
- 생성된 차트의 제목을 수정합니다 (예: "실험 2차 결과 분석").
- 차트 하단에 "이 그래프는 압력과 온도의 상관관계를 보여줍니다"라는 설명을 추가합니다.
- 배경색이나 폰트를 연구실 브랜드 가이드에 맞춰 조정합니다.

#### 3.2.4 배포 및 활용
- 'Publish' 버튼을 눌러 고유 URL을 생성합니다.
- 생성된 URL을 팀원들에게 공유하여 피드백을 받습니다.
- 'Export to PNG'로 고해상도 이미지를 내려받아 논문에 삽입합니다.

### 4.2 Phase 5 과제 안내 (15분)

이번 회차의 핵심은 '직접 도구를 만들어 보는 것'입니다.

#### 4.2.1 과제 배경: 왜 '디스포저블 앱'인가?
우리는 매번 거창한 소프트웨어를 개발할 필요가 없습니다. 특정 문제를 해결하기 위해 30분 만에 뚝딱 만들어 쓰고, 목적을 달성하면 미련 없이 버릴 수 있는 앱이 연구 효율을 극대화합니다.

#### 4.2.2 과제 수행 가이드
- **주제 선정**: "매번 엑셀에서 하는 이 계산이 귀찮다", "이 로그 파일을 보기 좋게 시각화하고 싶다"는 생각이 드는 지점을 찾으세요.
- **도구 선택**: nanobanana-pro, Claude Artifacts, Streamlit 중 본인에게 편한 것을 고르세요.
- **기록**: 제작 과정에서 AI에게 던진 프롬프트와, 실제 제작에 소요된 시간을 반드시 기록하세요.

## 📝 오늘 배운 내용 정리

### 핵심 요약

1. **AI 프레젠테이션**: 디자인 노동에서 해방되어 **구조(Structure)**와 **메시지(Message)**에 집중하는 법을 배웠습니다.
2. **Claude for Excel**: 엑셀을 단순한 표 계산기가 아닌, AI와 대화하며 데이터를 요리하는 **분석 도구**로 재정의했습니다.
3. **nanobanana-pro**: 코딩 없이 전문가 수준의 인터랙티브 시각화 구현을 확인했습니다.
4. **디스포저블 앱**: 필요할 때 즉시 만들어 쓰는 **Flywheel 사이클**의 첫 단추를 끼웠습니다.

### 자주 묻는 질문 (Q&A)

**Q: AI가 만든 수식이 틀리면 어떻게 하나요?**
A: AI는 확률적으로 답변하므로 오류가 있을 수 있습니다. 반드시 샘플 데이터로 검증하고, 오류 발생 시 에러 메시지를 그대로 Claude에게 전달하여 수정을 요청하세요.

**Q: nanobanana 무료 버전의 제한은 무엇인가요?**
A: 업로드 가능한 데이터 용량과 생성 가능한 대시보드 개수에 제한이 있을 수 있습니다. 연구실 단위로 사용한다면 유료 플랜을 고려하거나, 오픈소스 대안인 Streamlit을 학습하는 것도 좋은 방법입니다.

**Q: 발표자료에 AI가 만든 이미지를 써도 저작권 문제가 없나요?**
A: 대부분의 AI 도구는 생성된 이미지에 대한 상업적 이용 권한을 부여하지만, 학회나 기관의 규정에 따라 AI 사용 여부를 명시해야 할 수도 있습니다.

## 🛠️ 문제 해결 가이드 (Troubleshooting)

### 1. Gamma.app에서 한글 폰트가 깨질 때
- 설정에서 폰트를 'Noto Sans KR' 또는 'Pretendard'와 같은 한글 지원 폰트로 변경하세요.
- 텍스트 박스의 너비를 조정하여 글자가 잘리지 않게 하세요.

### 2. 엑셀 VBA 코드가 실행되지 않을 때
- [파일] -> [옵션] -> [보안 센터] -> [매크로 설정]에서 '모든 매크로 포함'이 선택되어 있는지 확인하세요.
- 코드 내의 시트 이름이 실제 엑셀 시트 이름과 일치하는지 확인하세요.

### 3. nanobanana에서 데이터 업로드 에러가 날 때
- 파일 형식이 CSV(UTF-8)인지 확인하세요.
- 데이터에 특수문자나 빈 행이 너무 많은지 확인하고 정제 후 다시 시도하세요.

## 🚀 심화 학습: Python과 Streamlit을 활용한 커스텀 앱 제작

더 높은 자유도를 원하는 연구자를 위해 Python 기반의 웹 프레임워크인 Streamlit을 소개합니다.

### 1. Streamlit이란?
Python 코드 몇 줄만으로 데이터 분석 웹 앱을 만들 수 있는 도구입니다. nanobanana보다 복잡한 로직을 구현할 때 유용합니다.

### 2. Claude와 함께하는 Streamlit 개발
- **요청**: "Python과 Streamlit을 사용해서 CSV 파일을 업로드하면 상관관계 행렬(Correlation Matrix)을 그려주는 앱 코드를 작성해줘."
- **실행**: Claude가 준 코드를 `app.py`로 저장하고 `streamlit run app.py` 명령어로 실행합니다.

### 3. 연구용 앱 예시
- **논문 요약기**: PDF를 업로드하면 Claude API를 호출해 핵심 내용을 요약해주는 앱
- **실험 파라미터 계산기**: 복잡한 물리 공식을 입력해두고 변수만 바꿔가며 결과를 확인하는 앱

## 📊 데이터 시각화 베스트 프랙티스 (Best Practices)

AI가 차트를 그려주더라도, 연구자로서 좋은 시각화의 원칙을 알고 있어야 합니다.

1. **단순함의 미학**: 불필요한 격자선, 화려한 3D 효과는 데이터 해석을 방해합니다.
2. **색상의 의미**: 강조하고 싶은 데이터에만 강한 색상을 사용하세요.
3. **축의 정직함**: Y축의 시작점을 0으로 설정하여 데이터의 차이를 왜곡하지 마세요.
4. **캡션의 중요성**: 차트만 보고도 결론을 알 수 있도록 명확한 제목과 설명을 다세요.
