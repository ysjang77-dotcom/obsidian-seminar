## Part 1: 세미나 철학과 AI 시대 패러다임 (40분)

### 1.1 왜 "퇴직준비"인가? (15분)

#### 핵심 메시지

> **"퇴직 = 회사를 떠나는 것"이 아닙니다.**  
> **"퇴직 = 내가 현재 수행하는 '직무'의 소멸에 대비하는 것"입니다.**

AI 이후 시대의 '직'은 현재의 형태가 아닐 것이 확실합니다. 시대는 결국 우리를 해고해버릴 것이나, **우리가 먼저 준비한다면 능히 대처**해나갈 수 있습니다.

#### 토론 질문

```
💬 "5년 후, 여러분의 현재 업무 중 AI가 대체할 수 있는 부분은 무엇이라고 생각하시나요?"
```

#### 사라질 것 vs 생존 스킬

| 사라질 것 | 생존 스킬 |
|-----------|-----------|
| 근무시간 | **상시 협업 체계** (AI Agent 24/7) |
| 문서작성 | **지식 큐레이션** (구조화된 사고 → 자동 문서화) |
| 연구실 | **개인 연구 인프라** (디스포저블 앱, 연구 에이전트) |
| 관리체계 | **자기 조직화** (Vibe Kanban, 개인 태스크 오케스트레이션) |
| 직무 | **역할 유동성** (T자형 역량 확장) |
| 전문가 의존 | **AI 1차 처리 + 전문가 검증 체계** |
| 정보 검색 | **Context-aware 정보 접근** (RAG, MCP) |

### 1.2 AI 시대 연구자의 역할 변화 (15분)

#### 기존 패러다임 vs 새로운 패러다임

| 구분 | 기존 | AI 시대 |
|------|------|---------|
| **지식 습득** | 직접 학습, 암기 | AI에게 질문, 검증 능력 |
| **문서 작성** | 처음부터 직접 작성 | AI 초안 → 인간 검토/수정 |
| **코딩** | 전문가만 가능 | 누구나 AI와 함께 가능 |
| **데이터 분석** | 전문 도구 학습 필요 | 자연어로 분석 요청 |
| **연구 조사** | 수동 검색, 정리 | AI 에이전트 활용 |
| **역할** | 실행자 (Doer) | **오케스트레이터 (Orchestrator)** |

#### 핵심 역량 변화

> **"알고 있는 것"에서 "질문하는 능력"으로**

```
과거: 정답을 알고 있어야 했다
현재: 올바른 질문을 할 수 있어야 한다
미래: AI와 협업하여 결과를 검증할 수 있어야 한다
```

## Part 2: PARA 폴더 구조 생성 실습 (40분)

### 2.1 PARA 방법론이란? (10분)

**PARA**는 Tiago Forte가 개발한 디지털 정보 관리 시스템입니다.

| 폴더 | 의미 | 설명 |
|------|------|------|
| **P**rojects | 프로젝트 | 명확한 목표와 마감일이 있는 진행 중인 작업 |
| **A**reas | 영역 | 지속적으로 관리해야 하는 책임 영역 |
| **R**esources | 자원 | 나중에 참고할 수 있는 주제별 자료 |
| **A**rchive | 보관함 | 완료되었거나 비활성화된 항목 |

#### 왜 PARA인가?

```
❌ 기존 방식: 주제별 분류 (생물학/화학/물리학...)
   → 문제: "이 자료가 어디에 속하지?" 결정 장애

✅ PARA 방식: 행동 기반 분류 (지금 작업 중/계속 관리/나중에 참고/끝남)
   → 장점: "이 자료로 지금 뭘 할 건가?"에 따라 분류
```

---

### 2.2 실습: PARA 폴더 구조 생성 (30분)

#### Step 1: 기본 폴더 생성

Obsidian Vault 루트에 다음 폴더를 생성하세요:

```
📁 내 Vault
├── 📂 Projects/       # 진행 중인 프로젝트
├── 📂 Areas/          # 지속 관리 영역
├── 📂 Resources/      # 참고 자료
├── 📂 Archive/        # 완료/비활성
└── 📂 Templates/      # 템플릿 (선택)
```

>[!tip] **Obsidian에서 폴더 만들기**
> 1. 좌측 파일 탐색기에서 우클릭
> 2. "New folder" 선택
> 3. 폴더 이름 입력

#### Step 2: 샘플 노트 생성

각 폴더에 최소 1개의 노트를 만들어보세요:

| 폴더 | 샘플 노트 예시 |
|------|---------------|
| Projects/ | `퇴직준비세미나 학습.md` |
| Areas/ | `건강관리.md` 또는 `자기계발.md` |
| Resources/ | `AI 도구 모음.md` 또는 `유용한 링크.md` |
| Archive/ | (아직 비워두기) |

#### Step 3: 노트에 내용 추가

`Projects/퇴직준비세미나 학습.md` 예시:

```markdown
# 퇴직준비세미나 학습

## 목표
- AI 시대 생존 스킬 습득
- Obsidian + Claude Code + Git 워크플로우 구축

## 진행 현황
- [x] Phase 1 사전과제 완료
- [ ] 1회차 세미나 참석
- [ ] PARA 구조 완성

## 메모
- 오늘 배운 내용: ...
```

#### Step 4: 양방향 링크 연습

노트 내에서 `[[다른 노트 이름]]` 형식으로 링크를 만들어보세요:

```markdown
이 내용은 [[AI 도구 모음]]과 관련이 있습니다.
```

>[!info] **양방향 링크의 힘**
> A 노트에서 B를 링크하면, B 노트의 "Backlinks" 패널에서 A가 자동으로 표시됩니다.
> 이것이 Obsidian의 핵심 기능입니다!

## Part 3: Claude Code 인증 및 첫 대화 (40분)

### 3.1 Claude Code란? (5분)

Claude Code는 **터미널에서 실행되는 AI 코딩 어시스턴트**입니다.

| 특징 | 설명 |
|------|------|
| **터미널 기반** | GUI가 아닌 CLI로 동작 |
| **프로젝트 인식** | 현재 폴더의 파일들을 이해 |
| **파일 조작** | 파일 읽기/쓰기/수정 가능 |
| **Git 연동** | 커밋, 브랜치 등 Git 작업 대행 |
| **자연어 명령** | "이 파일 수정해줘"처럼 대화체로 요청 |

>[!warning] **Claude Pro 구독 필요**
> Claude Code는 월 $20의 Claude Pro 구독이 필요합니다.
> 아직 구독하지 않으셨다면 [claude.ai](https://claude.ai)에서 업그레이드하세요.

---

### 3.2 실습: Claude Code 실행 및 인증 확인 (10분)

#### Step 1: 터미널 열기

**Windows**: Git Bash 또는 PowerShell
**Mac**: Terminal

#### Step 2: 설치 확인

```bash
claude --version
```

정상 출력 예시:
```
Claude Code version 1.x.x
```

#### Step 3: 진단 실행

```bash
claude doctor
```

모든 항목이 ✅ 체크되어야 합니다.

#### Step 4: Obsidian Vault 폴더로 이동

```bash
cd ~/Documents/내Vault폴더
```

---

### 3.3 실습: Claude Code 첫 대화 (25분)

#### Step 1: Claude Code 시작

```bash
claude
```

#### Step 2: 첫 대화 - 폴더 탐색

```
> 이 폴더의 구조를 설명해줘
```

Claude가 현재 폴더의 파일들을 분석하고 설명합니다.

#### Step 3: 파일 생성 요청

```
> Resources 폴더에 "Claude Code 사용법.md" 파일을 만들어줘.
> 내용은 오늘 배운 Claude Code 기본 명령어를 정리해줘.
```

>[!tip] **Claude Code 기본 명령어**
> - `/help` - 도움말 보기
> - `/init` - CLAUDE.md 파일 생성 (프로젝트 컨텍스트)
> - `/config` - 설정 확인/변경
> - `Shift+Tab` - Plan 모드 전환
> - `/clear` - 대화 초기화

#### Step 4: 프로젝트 초기화

```
> /init
```

이 명령어는 현재 폴더에 `CLAUDE.md` 파일을 생성합니다.
이 파일은 Claude가 프로젝트를 이해하는 데 도움이 됩니다.

## Part 4: Git 워크플로우 + 정리 (30분)

### 4.1 AI와 함께하는 Git (15분)

>[!info] **핵심 원칙**
> Git 명령어를 외울 필요 없습니다. Claude Code에게 자연어로 요청하세요!

#### Claude Code로 Git 사용하기

| 하고 싶은 일 | Claude에게 요청 |
|-------------|----------------|
| 변경사항 확인 | "지금 뭐가 변경됐는지 보여줘" |
| 커밋하기 | "변경사항 커밋해줘" |
| 원격에 업로드 | "GitHub에 푸시해줘" |
| 최신 내용 받기 | "원격에서 최신 내용 가져와줘" |

#### 실습: Claude Code로 커밋하기

```
> git status 확인하고, 변경사항을 적절한 메시지로 커밋해줘
```

>[!warning] **주의사항**
> - Push 전에는 반드시 내용을 확인하세요
> - 민감한 정보(.env, 비밀번호 등)는 커밋하지 마세요
> - `.gitignore`를 잘 설정해두세요

자세한 내용: [[000_PARA/Resource/008_eduDoc/버전관리/AI 도구와 Git 관리]]
