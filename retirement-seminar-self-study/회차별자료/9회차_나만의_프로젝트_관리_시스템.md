## Part 1: 멀티 에이전트 오케스트레이션 개념 (40분)

### 1.1 왜 멀티 에이전트인가? (10분)

지금까지 우리는 Claude나 Claude Code와 1:1로 대화하며 작업을 수행했습니다. 하지만 실제 프로젝트는 여러 개의 복잡한 태스크가 얽혀 있습니다.

- **한계**: 한 번에 하나의 작업만 수행 가능, 문맥(Context)의 비대화, 작업 간 전환 비용 발생
- **해결책**: 작업을 쪼개고, 각 작업에 독립적인 AI 에이전트를 할당하여 병렬로 처리

> [!info] **핵심 인사이트**
> "나 = PM + 개발자 + QA"
> 당신은 더 이상 코드를 직접 짜는 사람이 아니라, 태스크를 정의하고 AI 에이전트들에게 업무를 배분하는 **프로젝트 매니저(PM)**입니다.

### 1.2 오케스트레이션(Orchestration)이란? (15분)

여러 명의 연주자(AI 에이전트)가 하나의 곡(프로젝트)을 완성하기 위해 조화롭게 연주하도록 지휘하는 과정입니다.

1. **태스크 분해 (Decomposition)**: 큰 목표를 실행 가능한 작은 단위로 나눕니다.
2. **에이전트 할당 (Assignment)**: 각 태스크에 적합한 에이전트(Claude Code, OpenCode 등)를 배정합니다.
3. **병렬 실행 (Parallel Execution)**: 에이전트들이 서로 간섭 없이 동시에 작업을 수행합니다.
4. **결과 통합 (Integration)**: 각 에이전트의 결과물을 검토하고 메인 프로젝트에 합칩니다.

### 1.3 멀티 에이전트 환경의 도전 과제 (15분)

- **코드 충돌**: 여러 에이전트가 같은 파일을 수정할 때 발생하는 문제
- **상태 관리**: 어떤 에이전트가 어디까지 작업했는지 추적하기 어려움
- **리뷰 부하**: 10명분 일을 하는 AI의 결과물을 혼자서 다 검토해야 함

이러한 문제들을 해결하기 위해 등장한 도구가 바로 **Vibe Kanban**입니다.

## Part 2: Vibe Kanban 핵심 기능과 Git Worktree (40분)

### 2.1 Vibe Kanban 아키텍처 (10분)

Vibe Kanban은 AI 코딩 에이전트를 팀처럼 관리할 수 있는 시각적 대시보드입니다.

```
┌─────────────────────────────────────────┐
│            Vibe Kanban UI               │
│         (localhost:3000)                │
├─────────────────────────────────────────┤
│           Backend Server                │
│    - Task Management (Kanban)           │
│    - Git Worktree Management            │
│    - Agent Orchestration                │
├─────────────────────────────────────────┤
│         AI Coding Agents                │
│  ┌─────┐ ┌─────┐ ┌─────┐              │
│  │Claude│ │Codex│ │ Amp │  ...        │
│  │ Code │ │     │ │     │              │
│  └─────┘ └─────┘ └─────┘              │
└─────────────────────────────────────────┘
```

### 2.2 핵심 기능 1: Git Worktree 기반 병렬 작업 (15분)

Vibe Kanban의 가장 강력한 기능은 **Git Worktree**를 자동으로 관리한다는 점입니다.

- **기존 방식**: `git checkout`으로 브랜치를 옮겨 다니며 작업 (한 번에 한 브랜치만 가능)
- **Worktree 방식**: 각 브랜치를 별도의 폴더로 꺼내어 동시에 작업 가능

```bash
# Vibe Kanban이 내부적으로 수행하는 작업
# 각 태스크마다 독립적인 물리 폴더(worktree) 생성
git worktree add ../task-123-feature main
```

이 방식을 통해 AI 에이전트 A는 `folder_A`에서, 에이전트 B는 `folder_B`에서 서로의 작업을 방해하지 않고 동시에 코딩할 수 있습니다.

### 2.3 핵심 기능 2: 시각적 칸반 보드 (15분)

- **Todo**: 대기 중인 태스크
- **In Progress**: AI 에이전트가 할당되어 작업 중인 상태 (실시간 로그 확인 가능)
- **Review**: 작업 완료 후 사용자의 승인을 기다리는 상태
- **Done**: 머지 완료

> [!tip] **Vibe Kanban 실행하기**
> 터미널에서 `npx vibe-kanban`을 입력하면 즉시 로컬 서버가 실행되고 브라우저에서 관리 화면이 열립니다.

## Part 3: 태스크→AI 할당→브랜치→리뷰→PR 플로우 (40분)

### 3.1 실습 준비: 프로젝트 연결 (10분)

1. `npx vibe-kanban` 실행
2. "+ New Project" 클릭
3. 로컬 Git 저장소 경로 선택
4. 프로젝트 설정 완료

### 3.2 태스크 등록 및 에이전트 할당 (10분)

- **태스크 등록**: 요구사항을 명확히 작성합니다. (Part 1의 프롬프트 기법 활용)
- **에이전트 선택**: 해당 태스크를 수행할 AI 모델을 선택합니다.
- **실행**: "Run" 버튼을 누르면 Vibe Kanban이 자동으로 브랜치를 만들고 에이전트를 구동합니다.

### 3.3 실시간 모니터링 및 리뷰 (10분)

- **로그 추적**: 에이전트가 어떤 파일을 수정하고 어떤 명령어를 실행하는지 실시간으로 봅니다.
- **Diff 확인**: 작업이 끝나면 변경된 코드의 차이점(Diff)을 시각적으로 확인합니다.

### 3.4 PR 생성 및 머지 (10분)

- **승인(Approve)**: 코드가 만족스러우면 승인합니다.
- **머지(Merge)**: 메인 브랜치로 코드를 합칩니다. Vibe Kanban은 이 과정을 자동화하여 Git 명령어를 직접 칠 필요가 없게 해줍니다.

## Part 4: Vibe Kanban 멀티 에이전트 실습 (30분)

### 4.1 워크숍: 3개 태스크 동시 처리하기 (20분)

본인의 프로젝트에서 다음 3가지 작업을 동시에 수행해봅니다.

1. **문서화**: README.md 개선 및 API 문서 생성
2. **기능 구현**: 간단한 유틸리티 함수 추가
3. **리팩토링**: 기존 코드의 가독성 개선

**실습 순서**:
1. 3개의 태스크를 칸반 보드에 등록합니다.
2. 각각에 에이전트를 할당하고 실행합니다.
3. 3개의 에이전트가 동시에 돌아가는 모습을 관찰합니다.
4. 완료된 순서대로 리뷰하고 머지합니다.

### 4.2 트러블슈팅 및 팁 (10분)

- **에이전트가 멈췄을 때**: 로그를 확인하고 필요한 경우 수동으로 개입하거나 재시작합니다.
- **의존성 문제**: A 태스크가 끝나야 B 태스크를 할당할 수 있는 경우, 칸반의 순서를 조정합니다.
- **Git 충돌**: 드물게 발생하는 충돌은 Vibe Kanban의 Diff 뷰에서 직접 수정할 수 있습니다.

## Appendix: 심화 학습 가이드

### A.1 Git Worktree 수동 관리하기
Vibe Kanban 없이도 Worktree를 사용할 수 있습니다.
```bash
# 새 워크트리 추가
git worktree add ../my-new-feature-folder feature-branch

# 워크트리 목록 확인
git worktree list

# 워크트리 삭제
git worktree remove ../my-new-feature-folder
```

### A.2 MCP(Model Context Protocol)와 Vibe Kanban
Vibe Kanban은 MCP 서버로도 동작할 수 있어, Claude Desktop 등 다른 도구에서도 칸반 보드의 태스크를 조회하고 관리할 수 있습니다.

### A.3 에이전트 성능 비교
- **Claude Code**: 복잡한 로직 구현 및 리팩토링에 강점
- **OpenCode**: 파일 생성 및 간단한 스크립트 작성에 최적화
- **Amp**: 대규모 프로젝트의 구조 파악 및 탐색에 유리

### A.4 프로젝트 매니징 팁
- **태스크 크기**: 에이전트가 10분 내외로 끝낼 수 있는 크기가 가장 좋습니다.
- **명확한 정의**: "코드를 고쳐줘" 보다는 "X 파일의 Y 함수에서 Z 에러를 수정하고 테스트 코드를 작성해줘"가 훨씬 효과적입니다.
- **리뷰 습관**: AI가 짠 코드는 반드시 `View Diff`를 통해 눈으로 확인하는 습관을 들여야 합니다.

### A.5 실전 사례: 1인 개발자의 하루
1. 아침에 5개의 태스크를 칸반에 등록합니다.
2. 5명의 에이전트에게 각각 할당합니다.
3. 커피를 마시며 에이전트들의 로그를 모니터링합니다.
4. 완료된 태스크들을 하나씩 리뷰하고 머지합니다.
5. 오전 중에 하루치 개발 업무를 모두 끝냅니다.

## B. 자주 묻는 질문 (FAQ)

**Q: Vibe Kanban은 유료인가요?**
A: 기본적으로 오픈소스 도구이며, 로컬에서 무료로 사용할 수 있습니다. 다만 사용하는 AI 모델(Claude 등)의 API 비용은 별도로 발생합니다.

**Q: 인터넷 연결이 없어도 사용 가능한가요?**
A: UI와 서버는 로컬에서 돌지만, AI 에이전트가 클라우드 모델(Claude 등)을 사용한다면 인터넷 연결이 필요합니다. 로컬 모델(Llama 등)을 연결한다면 오프라인에서도 가능합니다.

**Q: 기존에 쓰던 Jira나 Trello와 연동되나요?**
A: 현재는 로컬 Git 저장소 중심의 독립적인 시스템입니다. 하지만 API나 MCP를 통해 커스텀 연동을 구현할 수 있습니다.

**Q: 코드가 유출될 걱정은 없나요?**
A: 모든 작업은 본인의 로컬 컴퓨터(Git Worktree)에서 이루어집니다. AI 모델에게 전달되는 데이터는 설정한 프롬프트와 관련 코드 조각뿐입니다.

## F. Vibe Kanban 실전 워크플로우 상세 가이드

### F.1 단계별 실행 시나리오 (Step-by-Step)

#### 1단계: 프로젝트 초기화 및 저장소 연결
Vibe Kanban은 Git 저장소를 기반으로 동작합니다. 기존 프로젝트가 없다면 새로 생성합니다.
```bash
mkdir my-ai-team-project
cd my-ai-team-project
git init
echo "# My AI Team Project" > README.md
git add .
git commit -m "initial commit"
```
그 후 `npx vibe-kanban`을 실행하고 브라우저에서 해당 폴더를 프로젝트로 등록합니다.

#### 2단계: 백로그(Backlog) 작성 및 태스크 세분화
PM의 관점에서 프로젝트에 필요한 기능들을 나열합니다.
- 예: "사용자 로그인 기능 구현", "데이터베이스 스키마 설계", "프론트엔드 메인 페이지 레이아웃"
- 각 태스크는 독립적으로 실행 가능하도록 최대한 작게 쪼개는 것이 핵심입니다.

#### 3단계: 에이전트 페르소나 설정
Vibe Kanban 설정에서 각 에이전트에게 특화된 역할을 부여할 수 있습니다.
- **에이전트 A (Architect)**: 전체적인 구조 설계 및 파일 레이아웃 담당
- **에이전트 B (Developer)**: 구체적인 로직 구현 및 함수 작성 담당
- **에이전트 C (QA/Tester)**: 테스트 코드 작성 및 버그 수정 담당

#### 4단계: 병렬 실행 및 모니터링
칸반 보드에서 태스크를 드래그하여 에이전트에게 할당합니다.
- 에이전트들이 각자의 Worktree에서 작업을 시작하면, 대시보드에서 실시간으로 진행률과 로그를 확인합니다.
- 특정 에이전트가 막히면(Stuck), 해당 Worktree 폴더로 직접 들어가서 코드를 수정하거나 힌트를 줄 수 있습니다.

#### 5단계: 통합 리뷰 및 품질 관리
작업이 완료된 태스크들은 `Review` 열로 이동합니다.
- `View Diff` 기능을 통해 변경된 모든 파일을 한눈에 확인합니다.
- `npx vibe-kanban-review` 명령어를 사용하여 AI에게 다시 한번 코드 리뷰를 맡길 수도 있습니다.
- 모든 검증이 끝나면 `Merge` 버튼을 눌러 메인 브랜치에 반영합니다.

## G. 1인 기업가를 위한 멀티 에이전트 전략

### G.1 비용 효율적인 에이전트 운영
모든 작업에 가장 비싼 모델(Claude 3.5 Sonnet 등)을 쓸 필요는 없습니다.
- **단순 반복 작업**: 하위 모델(Claude 3 Haiku 등)을 사용하여 비용 절감
- **복잡한 로직/설계**: 상위 모델을 사용하여 품질 확보
- **문서 번역/정리**: 무료 모델이나 저가형 모델 활용

### G.2 에이전트 간 협업 패턴
- **릴레이 패턴**: 에이전트 A가 초안을 작성하면, 에이전트 B가 이를 검토하고 보완하는 방식
- **경쟁 패턴**: 같은 태스크를 두 명의 에이전트에게 맡기고 더 좋은 결과물을 선택하는 방식
- **교차 리뷰**: 에이전트 A의 코드를 에이전트 B가 테스트하게 하는 방식

### G.3 지식 자산화 (Knowledge Asset)
에이전트들이 생산한 결과물뿐만 아니라, 그 과정에서 사용된 **프롬프트와 워크플로우**를 기록해두는 것이 중요합니다.
- Obsidian의 PARA 시스템을 활용하여 성공적인 에이전트 지시문(Instruction)을 저장하세요.
- 다음 프로젝트에서 이를 재사용하여 시작 시간을 단축할 수 있습니다.

## O. 향후 전망: 에이전틱 워크플로우(Agentic Workflow)의 미래

우리가 오늘 배운 Vibe Kanban은 시작에 불과합니다. 앞으로의 업무 환경은 다음과 같이 변할 것입니다.

1. **자율적 에이전트**: 사람이 태스크를 일일이 지정하지 않아도, 목표(Goal)만 주면 에이전트들이 스스로 태스크를 나누고 협업하는 단계로 진화할 것입니다.
2. **에이전트 마켓플레이스**: 특정 분야(법률, 회계, 특허, 설계)에 특화된 전문 에이전트들을 고용하여 팀을 구성하는 시대가 올 것입니다.
3. **개인화된 에이전트**: 나의 업무 스타일, 과거 작업 기록, 선호하는 코드 스타일을 학습한 '나만의 분신' 에이전트가 탄생할 것입니다.

이러한 미래에서 가장 중요한 역량은 기술 그 자체가 아니라, **"무엇을 만들 것인가"를 결정하는 기획력**과 **"어떻게 협업할 것인가"를 설계하는 오케스트레이션 능력**입니다.
