## Phase 1-2 세미나 소개

Phase 1-2의 주제는 **'근무시간이 사라진다 - 상시 협업 체계 구축'**입니다.

> "AI 에이전트는 단순히 대화하는 상대가 아닙니다.  
> 스스로 생각하고, 도구를 사용하며, 목표를 달성할 때까지 멈추지 않는 당신의 디지털 분신입니다."

Phase 1-2에서는 AI Agent의 핵심 개념을 배우고, 효과적인 프롬프트 작성법을 학습합니다.

## 왜 Phase 1-2 과제가 도전적인가?

### 1. 프롬프트 설계 능력이 필요합니다

단순히 AI에게 질문하는 것이 아니라, **체계적인 지시**를 내려야 합니다.
- 역할(Role) + 맥락(Context) + 지시(Instruction) + 형식(Format)의 골든 공식
- 모호한 지시 vs 명확한 지시의 차이 체감

### 2. Hallucination 검증 능력이 필요합니다

AI의 답변을 그대로 믿으면 안 됩니다.
- AI가 생성한 정보의 사실 여부를 교차 검증하는 습관
- 출처를 확인하고 오류를 발견하는 능력

### 3. 반복적인 개선이 필요합니다

한 번에 완벽한 프롬프트를 작성하기 어렵습니다.
- v1 → v2 → v3로 개선해가는 과정
- 왜 수정했는지 기록하고 분석하는 능력

### 프롬프트 실전 과제

프롬프트 골든 공식을 적용하여 실제 업무 문서를 작성하고 AI 답변을 검증합니다.

| 항목 | 설명 |
|------|------|
| Claude로 업무 문서 1건 작성 | 회의록, 보고서, 기획서, 메일 초안 중 택1 |
| 프롬프트 골든 공식 적용 | 역할 + 맥락 + 지시 + 형식 4요소 포함 |
| Hallucination 검증 (최소 3건) | AI 답변의 사실 정보를 교차 검증 |
| 프롬프트 개선 히스토리 | v1 → v2 → v3 (최소 2회 수정) |

### 제출물

1. **최종 프롬프트** (마크다운 파일)
2. **프롬프트 개선 히스토리** (v1→v2→v3 변경 사항 및 이유)
3. **Hallucination 검증표** (3건 이상, 출처 명시)
4. **AI 결과물** (생성된 문서)

## 과제의 의미

Phase 1-2 과제를 통해 여러분은:

- [ ] AI를 단순한 챗봇이 아닌 **협업 파트너**로 인식하는 전환
- [ ] 프롬프트 설계를 통해 **AI의 행동을 제어**하는 경험
- [ ] AI 답변을 **검증하고 교정하는** 습관 형성
- [ ] 반복적 개선을 통해 **프롬프트 품질 향상** 체득

을 얻게 됩니다.

**이것이 바로 "24시간 협업 체계"의 첫걸음입니다.**
